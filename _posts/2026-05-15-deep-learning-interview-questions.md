大节）</li></ul><h1><strong>一、神经网络基础</strong></h1><p>本章涵盖神经网络的基础概念：激活函数、初始化、归一化、正则化、优化器、损失函数等核心内容。这些是所有深度学习模型（CNN、RNN、Transformer）的共同基石。</p><h2><strong>1.1 各个激活函数的优缺点</strong></h2><p>激活函数为神经网络引入非线性，使其能够拟合任意复杂函数。若没有非线性激活，无论叠加多少层，整个网络仍等价于一个线性变换。</p><p><img src="/assets/img/dl-interview/img-01.png" /></p><p><em>图1-1  常见激活函数及其导数（蓝色为函数本身，红色虚线为导数）</em></p><h3><strong>Sigmoid</strong></h3><p>σ(x) = 1 / (1 + e^(-x)),    σ'(x) = σ(x)(1 - σ(x))</p><p><strong>优点：</strong></p><ul><li>输出在 (0, 1) 之间，可直接解释为概率，常用于二分类输出层。</li><li>函数处处连续可导，便于反向传播。</li></ul><p><strong>缺点：</strong></p><ul><li>梯度消失：当 |x| 较大时 σ'(x) → 0，深层网络反向传播时梯度被压缩到极小（最大值仅为 0.25）。</li><li>输出非零均值：所有激活值恒为正，导致反向传播中权重梯度始终同号，参数更新呈&quot;之&quot;字形震荡。</li><li>exp 运算计算开销较大。</li></ul><h3><strong>Tanh（双曲正切）</strong></h3><p>tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x)),    tanh'(x) = 1 - tanh²(x)</p><p><strong>优点：</strong></p><ul><li>输出在 (-1, 1) 之间，零均值，缓解了 Sigmoid 的偏移问题，收敛速度更快。</li><li>导数最大值为 1（在 x=0 处），梯度大于 Sigmoid。</li></ul><p><strong>缺点：</strong></p><ul><li>仍存在饱和区与梯度消失问题（|x| 较大时导数趋零）。</li><li>计算量与 Sigmoid 相当。</li></ul><h3><strong>ReLU（线性整流单元）</strong></h3><p>ReLU(x) = max(0, x)</p><p><strong>优点：</strong></p><ul><li>计算极快，仅一次比较运算。</li><li>正区间梯度恒为 1，有效缓解梯度消失。</li><li>生物学合理性：让神经元具有稀疏激活的特性。</li></ul><p><strong>缺点：</strong></p><ul><li>Dead ReLU 问题：若某神经元的预激活长期为负，则其梯度恒为 0，该神经元永远无法被激活更新。常见于学习率过大或初始化不当时。</li><li>输出非零均值。</li><li>在 x=0 处不可导（但实际工程中将其约定为 0 或 1）。</li></ul><h3><strong>Leaky ReLU / PReLU / RReLU</strong></h3><p>LeakyReLU(x) = x  if x &gt; 0 else αx  (α 通常取 0.01)</p><p>为负区间引入一个小斜率，缓解 Dead ReLU。</p><ul><li>Leaky ReLU：α 固定为小常数（0.01）。</li><li>PReLU：α 作为可学习参数，由网络自行训练。</li><li>RReLU：α 在训练时从均匀分布随机采样，测试时取期望。</li></ul><h3><strong>ELU / SELU</strong></h3><p>ELU(x) = x  if x &gt; 0 else α(e^x - 1)</p><p>负区间用指数曲线平滑过渡，输出均值更接近 0；SELU 在 ELU 基础上加自归一化系数，理论上可让深层网络自我维持零均值单位方差。</p><h3><strong>GELU / Swish</strong></h3><p>GELU(x) = x · Φ(x),    Swish(x) = x · σ(βx)</p><p>GELU 已成为 Transformer / BERT 的事实标准激活；Swish 由 Google 通过自动搜索得到，在多数 CNN 任务上略优于 ReLU。两者共同特点：平滑、非单调、允许小幅负值通过，组合了门控思想。</p><h2><strong>1.2 为什么 ReLU 常用于神经网络的激活函数</strong></h2><ol><li>缓解梯度消失：正区间梯度恒为 1，深层网络仍能保持梯度量级，使得训练 50 层以上的网络成为可能。</li><li>计算高效：相比 Sigmoid/Tanh 的指数运算，ReLU 仅需一次阈值比较，前向和反向都极快。</li><li>稀疏激活：约 50% 的神经元输出为 0，类似生物神经元的稀疏激发模式，提升表示效率并具备一定正则效果。</li><li>单边抑制：负输入直接归零，模型对噪声更鲁棒。</li><li>实践有效性：从 AlexNet（2012）以来，几乎所有 SOTA 视觉模型都默认使用 ReLU 或其变体。</li></ol><p><strong>💡 提示： </strong>ReLU 不是&quot;完美&quot;的——在 Transformer 等架构上 GELU 已取而代之。但作为通用默认值，ReLU 在工程上仍是最稳妥的选择。</p><h2><strong>1.3 梯度消失和梯度爆炸的解决方案？梯度爆炸引发的问题？</strong></h2><p>在深层网络的反向传播中，梯度需要经过多层连乘。若每层梯度 &lt; 1，相乘后趋向 0（梯度消失）；若每层梯度 &gt; 1，相乘后趋向无穷（梯度爆炸）。</p><p><img src="/assets/img/dl-interview/img-02.png" /></p><p><em>图1-2  Sigmoid 链式相乘导致梯度指数衰减；权重过大则梯度指数放大</em></p><p><strong>梯度消失的解决方案：</strong></p><ul><li>使用 ReLU 系列激活函数替换 Sigmoid/Tanh。</li><li>Batch Normalization：将每层激活拉回标准分布，避免落入饱和区。</li><li>残差连接（ResNet）：通过 y = F(x) + x 提供恒等映射，让梯度可直接回传到浅层。</li><li>合理初始化：He / Xavier 初始化使每层激活的方差保持稳定。</li><li>LSTM/GRU：通过门控机制为 RNN 提供&quot;梯度高速公路&quot;。</li></ul><p><strong>梯度爆炸的解决方案：</strong></p><ul><li>梯度裁剪（Gradient Clipping）：限制梯度的范数上限，是 RNN 训练的标配。</li><li>权重正则化（L1/L2）：约束权重不会变得过大。</li><li>合理初始化 + BN。</li><li>使用较小的学习率。</li></ul><p><strong>梯度爆炸引发的问题：</strong></p><ul><li>损失出现 NaN/Inf，训练崩溃。</li><li>权重更新步长过大，越过最优点，损失剧烈震荡。</li><li>模型完全失效，需重新训练。</li></ul><h2><strong>1.4 如何确定是否出现梯度爆炸</strong></h2><ul><li>损失值变为 NaN 或 Inf。</li><li>损失函数曲线剧烈震荡，无法收敛。</li><li>梯度范数（torch.nn.utils.clip_grad_norm_ 的返回值）持续 &gt; 1e2 或 1e3。</li><li>权重值在训练过程中迅速变大，甚至溢出。</li><li>模型预测全部输出极端值（如全 1 或全 0）。</li></ul><p>监控做法：训练时打印 grad_norm 与 weight_norm；TensorBoard 中观察每层的梯度直方图。</p><h2><strong>1.5 神经网络中有哪些正则化技术</strong></h2><p>正则化用于防止过拟合，让模型在测试集上的表现更好。</p><table><tr><td><p><strong>类别</strong></p></td><td><p><strong>方法</strong></p></td><td><p><strong>原理</strong></p></td></tr><tr><td><p>参数约束</p></td><td><p>L1 正则化</p></td><td><p>在损失中加入 λ‖w‖₁，促使权重稀疏，可做特征选择</p></td></tr><tr><td><p>参数约束</p></td><td><p>L2 正则化（权重衰减）</p></td><td><p>在损失中加入 λ‖w‖₂²，抑制权重过大，使决策面平滑</p></td></tr><tr><td><p>网络结构</p></td><td><p>Dropout</p></td><td><p>训练时随机让神经元失活，相当于集成大量子网络</p></td></tr><tr><td><p>网络结构</p></td><td><p>DropConnect</p></td><td><p>随机将连接的权重置零，比 Dropout 更细粒度</p></td></tr><tr><td><p>归一化</p></td><td><p>BN / LN / GN</p></td><td><p>约束每层激活的分布，间接起到正则作用</p></td></tr><tr><td><p>数据</p></td><td><p>数据增强</p></td><td><p>旋转、翻转、裁剪、Mixup、CutMix ε/(K-1)，防止过度自信</p></td></tr><tr><td><p>训练</p></td><td><p>权重平均（EMA / SWA）</p></td><td><p>对训练后期的权重做指数移动平均，提升泛化</p></td></tr></table><h2><strong>1.6 批量归一化（BN）如何实现？作用？</strong></h2><p>BN 由 Sergey Ioffe 在 2015 年提出，是深度学习训练最重要的技巧之一。</p><p><strong>实现步骤（对每个 mini-batch 的某一层激活）：</strong></p><p>μ_B = (1/m) Σ x_i</p><p>σ²_B = (1/m) Σ (x_i - μ_B)²</p><p>x̂_i = (x_i - μ_B) / sqrt(σ²_B + ε)</p><p>y_i = γ · x̂_i + β     (γ, β 为可学习的尺度和偏移参数)</p><p><img src="/assets/img/dl-interview/img-03.png" /></p><p><em>图1-3  BN 将激活分布标准化到 0 均值、单位方差</em></p><p><strong>作用：</strong></p><ul><li>加速收敛：允许使用更大的学习率，训练时间常缩短 5~10 倍。</li><li>缓解梯度消失：将激活拉回非饱和区。</li><li>一定的正则化效果：mini-batch 统计噪声相当于轻度数据增强，常可减少 Dropout 用量。</li><li>降低对初始化的敏感度。</li></ul><p>注意：测试阶段用整个训练集的运行均值（running mean）和运行方差代替 mini-batch 统计量。</p><h2><strong>1.7 神经网络中权值共享的理解</strong></h2><p>&quot;权值共享&quot;指同一组参数被网络中多个位置或多个时间步重复使用。</p><p><strong>典型场景：</strong></p><ul><li>CNN：同一个卷积核滑过整张特征图，所有位置共用一组权重。原因：自然图像的局部特征（边缘、纹理）具有平移不变性，无论目标出现在图像哪个位置，都应被同一组检测器识别。</li><li>RNN：所有时间步共用同一个权重矩阵 W_hh 与 W_xh。原因：序列数据的语法/语义规则不应依赖于绝对时间位置。</li></ul><p><strong>优点：</strong></p><ul><li>大幅减少参数量。例如 224×224 图像若用全连接需上亿参数，用 CNN 可降到百万级。</li><li>提供归纳偏置（inductive bias），使网络在小数据集上也能泛化。</li><li>使模型具备平移不变性 / 时间不变性。</li></ul><h2><strong>1.8 对 fine-tuning（微调模型）的理解？为什么要修改最后几层神经网络权值</strong></h2><p>Fine-tuning 是迁移学习的核心做法：先用大数据集（如 ImageNet）预训练得到通用特征提取器，再在目标小数据集上继续训练。</p><p><strong>为什么修改最后几层：</strong></p><ol><li>底层卷积学到的是低级通用特征（边缘、纹理、颜色），这些特征跨数据集通用，不需要重新学习。</li><li>高层（特别是分类头）学到的是任务相关的高级语义。新任务的类别空间不同，因此最后一层必须替换并重新训练。</li><li>小数据集上从头训练深网络极易过拟合，仅微调顶层可大幅缓解。</li></ol><h2><strong>1.9 什么是 Dropout？为什么有用？它是如何工作的</strong></h2><p>Dropout 由 Hinton 在 2012 年提出：训练时以概率 p 随机将神经元的输出置零，测试时使用全部神经元（但需缩放）。</p><p><img src="/assets/img/dl-interview/img-04.png" /></p><p><em>图1-4  训练时随机失活神经元；测试时使用完整网络</em></p><p><strong>工作原理：</strong></p><ol><li>训练时：每个 mini-batch 前，对每个隐层神经元独立地以概率 p 置零，反向传播只更新未被丢弃的连接。</li><li>测试时（Inverted Dropout 实现）：所有神经元都保留，但训练时输出已按 1/(1-p) 放大，因此测试时直接使用即可。</li></ol><p><strong>为什么有用：</strong></p><ul><li>集成视角：每次 mini-batch 都相当于训练一个不同的&quot;子网络&quot;，共享权重。最终模型相当于 2^N 个子模型的几何平均集成。</li><li>打破协同适应：神经元不能依赖某个固定的&quot;队友&quot;，被迫学习更鲁棒的特征。</li><li>增加输入噪声：相当于一种自适应的噪声注入。</li></ul><h2><strong>1.10 如何选择 dropout 的概率</strong></h2><ul><li>输入层：通常 p=0.1~0.2（不能丢太多原始信息）。</li><li>隐藏层：经典取值 p=0.5（Hinton 论文推荐）；现代网络配合 BN 后常用 0.1~0.3。</li><li>卷积层：很少使用普通 Dropout（卷积特征空间相关性强），改用 Spatial Dropout（整通道丢弃）或 DropBlock（连续区域丢弃）。</li><li>调参建议：模型容量大、过拟合严重时调高；模型欠拟合时调低或去掉。</li></ul><h2><strong>1.11 什么是 Adam？Adam 和 SGD 之间的主要区别</strong></h2><p>Adam（Adaptive Moment Estimation）由 Kingma 在 2014 年提出，结合了 Momentum（一阶矩）和 RMSProp（二阶矩）的思想。</p><p><strong>Adam 的更新规则：</strong></p><p>m_t = β₁·m_{t-1} + (1-β₁)·g_t                    (一阶矩, 类似动量)</p><p>v_t = β₂·v_{t-1} + (1-β₂)·g_t²                   (二阶矩, 梯度平方)</p><p>m̂_t = m_t / (1 - β₁^t),    v̂_t = v_t / (1 - β₂^t)   (偏差修正)</p><p>θ_t = θ_{t-1} - lr · m̂_t / (sqrt(v̂_t) + ε)</p><p>默认超参：β₁=0.9, β₂=0.999, ε=1e-8。</p><p><strong>Adam 与 SGD 的主要区别：</strong></p><table><tr><td><p><strong>对比维度</strong></p></td><td><p><strong>SGD（含 Momentum）</strong></p></td><td><p><strong>Adam</strong></p></td></tr><tr><td><p>学习率</p></td><td><p>全局统一</p></td><td><p>每个参数自适应</p></td></tr><tr><td><p>一阶矩</p></td><td><p>Momentum 维护</p></td><td><p>维护并做偏差修正</p></td></tr><tr><td><p>二阶矩（梯度平方）</p></td><td><p>不维护</p></td><td><p>维护，用于缩放学习率</p></td></tr><tr><td><p>收敛速度</p></td><td><p>前期慢，需精细调参</p></td><td><p>快，对初值不敏感</p></td></tr><tr><td><p>泛化能力</p></td><td><p>常常更好（尤其CV任务）</p></td><td><p>可能略差，可能陷入尖锐极小</p></td></tr><tr><td><p>内存开销</p></td><td><p>存 1 份动量</p></td><td><p>存 2 份矩估计</p></td></tr><tr><td><p>典型场景</p></td><td><p>ImageNet 分类、检测</p></td><td><p>NLP、GAN、小数据集快速实验</p></td></tr></table><h2><strong>1.12 为什么 Momentum 可以加速训练</strong></h2><p>v_t = γ·v_{t-1} + (1-γ)·g_t          θ_t = θ_{t-1} - lr · v_t</p><p>Momentum 引入物理学中的&quot;动量&quot;概念：当前更新方向不是当前梯度，而是历史梯度的加权累积。γ 通常取 0.9。</p><p><strong>加速原理：</strong></p><ol><li>在梯度方向一致的维度上（如长椭圆谷底的长轴方向），累积速度逐步加快，相当于动态放大学习率。</li><li>在梯度方向反复振荡的维度上（短轴方向），正负相消，抑制震荡。</li><li>帮助跨越平坦区域和小局部极小值（凭借惯性&quot;冲过去&quot;）。</li></ol><p><img src="/assets/img/dl-interview/img-05.png" /></p><p><em>图1-5  优化器收敛路径对比（左）；学习率影响（右）</em></p><h2><strong>1.13 什么时候使用 Adam 和 SGD</strong></h2><p><strong>Adam 适用场景：</strong></p><ul><li>快速原型验证：默认参数即可获得不错结果。</li><li>NLP 任务（Transformer、BERT 微调等）。</li><li>GAN、强化学习等损失非凸且复杂的场景。</li><li>数据稀疏（不同特征出现频率差异巨大）的场景。</li></ul><p><strong>SGD（Momentum / Nesterov）适用场景：</strong></p><ul><li>对最终精度有极致要求的视觉任务（ImageNet 分类、COCO 检测）。</li><li>需要精细控制收敛过程、配合学习率调度。</li><li>追求最优泛化能力，避免 Adam 陷入&quot;尖锐极小&quot;。</li></ul><p>实践折中：先用 Adam 快速训练若干 epoch，后期切换到 SGD 微调，常获两者之长。</p><h2><strong>1.14 batch size 和 epoch 的平衡</strong></h2><p>batch size：每次梯度更新所用样本数。epoch：整个训练集被遍历一次。</p><p><strong>batch size 的影响：</strong></p><ul><li>较大：梯度估计准，单步更新稳定，GPU 利用率高；但内存压力大、可能陷入尖锐极小、泛化变差。</li><li>较小：梯度有噪声（隐式正则），泛化可能更好；但训练慢、不稳定。</li></ul><p><strong>经验法则：</strong></p><ul><li>线性缩放规则（Goyal et al., 2017）：batch size 加倍时学习率也加倍，可在 ImageNet 上保持收敛性。</li><li>视觉任务常用 batch size 256；大模型预训练动辄数千甚至数万（配合梯度累积）。</li><li>小数据集（&lt;10k 样本）batch size 取 16~64。</li></ul><h2><strong>1.15 SGD 每步做什么，为什么能 online learning</strong></h2><p><strong>SGD 每步：</strong></p><ol><li>从训练集中采样一个（或一小批）样本。</li><li>前向计算损失。</li><li>反向传播得到梯度 g。</li><li>更新参数：θ ← θ - lr · g。</li></ol><p><strong>为什么支持 online learning：</strong></p><p>online learning 指数据按时间顺序到达，每次到一个样本就用它更新模型。SGD 本身的&quot;每次一个样本（或一小批）&quot;的更新机制天然契合这种场景——无需保留全部历史数据，可以边到达边学习。批量梯度下降（BGD）需要遍历全部样本才能更新一次，无法做 online。</p><h2><strong>1.16 学习率太大（太小）时会发生什么？如何设置学习率</strong></h2><p><strong>学习率太大：</strong></p><ul><li>损失剧烈震荡，甚至发散到 NaN。</li><li>在最优点附近来回跳，无法精确收敛。</li><li>梯度爆炸风险增加。</li></ul><p><strong>学习率太小：</strong></p><ul><li>收敛速度极慢，训练时间不可接受。</li><li>容易陷入糟糕的局部极小值或鞍点。</li></ul><p><strong>如何设置学习率：</strong></p><ol><li>LR Range Test（Smith, 2017）：从极小学习率开始，每个 batch 指数式增大，绘制 loss-lr 曲线，选择 loss 下降最快段对应的学习率作为最大值。</li><li>Warmup：训练初期用极小学习率（避免参数被错误大梯度带偏），然后线性升至目标值，再衰减。</li><li>调度策略：Step Decay、Cosine Annealing、Cyclical LR、ReduceLROnPlateau 等。</li><li>参考默认：Adam 常用 1e-3 ~ 1e-4；SGD+Momentum 常用 0.1 ~ 0.01（视觉分类）。</li></ol><h2><strong>1.17 神经网络为什么不用拟牛顿法而是用梯度下降</strong></h2><p>牛顿法用二阶导数信息（Hessian 矩阵）寻找极值，理论上收敛更快（二次收敛）。但在深度学习中几乎不用，原因：</p><ol><li>Hessian 计算/存储代价巨大：参数量为 N，Hessian 为 N×N，深度模型 N ≥ 10⁷，根本无法存储更别提求逆。</li><li>非凸性：神经网络损失曲面有大量鞍点和局部极小，Hessian 不一定正定，牛顿法可能朝错误方向更新。</li><li>mini-batch 噪声：Hessian 估计噪声大于梯度，得不偿失。</li><li>一阶方法+自适应+动量已经足够好：Adam、SGD+Momentum 在实践中收敛速度和泛化都已经令人满意。</li></ol><p>拟牛顿法（如 L-BFGS）在凸优化、小参数模型上仍有应用，但深度网络主流仍是一阶方法。</p><h2><strong>1.18 BN 和 Dropout 在训练和测试时的差别</strong></h2><table><tr><td></td><td><p><strong>训练时</strong></p></td><td><p><strong>测试时</strong></p></td></tr><tr><td><p>BN</p></td><td><p>使用当前 mini-batch 的均值/方差归一化，并更新运行均值/方差</p></td><td><p>使用累积的运行均值/方差归一化（固定值）</p></td></tr><tr><td><p>Dropout</p></td><td><p>以概率 p 随机失活神经元，输出按 1/(1-p) 放大（Inverted）</p></td><td><p>所有神经元都保留，不做任何随机操作</p></td></tr></table><p><strong>⚠️ 注意： </strong>PyTorch 中调用 model.eval() 即可切换 BN/Dropout 到测试模式，调用 model.train() 切回训练模式。常见 bug：忘记 eval() 导致测试结果异常。</p><h2><strong>1.19 若网络初始化为 0 的话有什么问题</strong></h2><p><strong>全零初始化导致对称性问题（symmetry problem）：</strong></p><ol><li>每个神经元接收相同输入、产生相同输出。</li><li>反向传播时每个神经元接收相同梯度，更新后权重仍然完全相同。</li><li>整层神经元始终保持完全一致，相当于只有一个神经元在工作。</li></ol><p>因此必须用随机初始化打破对称性。常用方案：Xavier（适合 Tanh/Sigmoid）、He/Kaiming（适合 ReLU 系）。偏置 b 可以初始化为 0，因为对称性已由权重打破。</p><h2><strong>1.20 sigmoid 和 softmax 的区别？softmax 的公式</strong></h2><p>sigmoid: σ(x) = 1 / (1 + e^(-x))            [输入一个标量，输出一个概率]</p><p>softmax: softmax(x)_i = e^(x_i) / Σ_j e^(x_j)   [输入向量，输出概率分布]</p><p><strong>区别：</strong></p><table><tr><td></td><td><p><strong>Sigmoid</strong></p></td><td><p><strong>Softmax</strong></p></td></tr><tr><td><p>输入</p></td><td><p>单个标量</p></td><td><p>向量（K维）</p></td></tr><tr><td><p>输出</p></td><td><p>一个 (0,1) 概率</p></td><td><p>K 维概率分布，和为 1</p></td></tr><tr><td><p>用途</p></td><td><p>二分类 / 多标签分类</p></td><td><p>互斥的多分类</p></td></tr><tr><td><p>类别独立性</p></td><td><p>各输出相互独立</p></td><td><p>各输出之间相互竞争（总和约束为 1）</p></td></tr></table><p><strong>📌 备注： </strong>特例：K=2 的 Softmax 与 Sigmoid 在数学上等价（只差一个常数偏置）。</p><h2><strong>1.21 改进的 softmax 损失函数有哪些</strong></h2><p>改进主要针对人脸识别等&quot;类间距离&quot;敏感任务：让同类样本更紧凑，不同类样本更分离。</p><ul><li>L-Softmax（Large-Margin Softmax）：在角度 θ 上引入乘性间隔 m，要求 cos(mθ) 才能正确分类。</li><li>A-Softmax / SphereFace：将权重归一化到单位球面上，直接学习角度判别。</li><li>AM-Softmax（CosFace）：在余弦相似度上做加法间隔，cos(θ) - m。</li><li>ArcFace：在角度上加加法间隔，cos(θ + m)，几何意义清晰，是人脸识别 SOTA。</li><li>Focal Loss（Lin et al., 2017）：解决类别不均衡，对易分类样本降权 (1-p)^γ。</li><li>Label Smoothing：将硬标签 [0,0,1,0] 改为 [ε/K, ε/K, 1-ε, ε/K]，缓解过度自信。</li></ul><h2><strong>1.22 深度学习调参有哪些技巧</strong></h2><ol><li>先过拟合一个小子集（如 100 样本），验证模型实现正确。</li><li>从已知有效的 baseline 出发，单次只改一个超参数。</li><li>学习率优先调：LR Range Test → 选最佳值 → 配合 warmup + cosine 衰减。</li><li>batch size 尽量大（在显存允许范围内），相应放大学习率。</li><li>用 Adam 快速验证想法，最终用 SGD+Momentum 拿到最佳数值。</li><li>weight decay 通常 1e-4 ~ 5e-4（视觉任务）；Adam 应使用 AdamW 的解耦权重衰减。</li><li>数据增强是免费的精度提升：CV 用 RandAugment/MixUp/CutMix；NLP 用 BackTranslation。</li><li>用 TensorBoard / Weights &amp; Biases 监控 loss、accuracy、梯度范数、权重直方图。</li><li>混合精度训练（fp16/bf16）可省一半显存并加速 1.5~2 倍。</li><li>随机种子固定，便于复现实验。</li></ol><h2><strong>1.23 神经网络调参，要往哪些方向想</strong></h2><p>遇到训练问题时，按照如下决策树排查：</p><ul><li>Loss 不下降 → 学习率是否过小？数据/标签是否正确？模型结构是否能拟合？</li><li>Loss 为 NaN → 学习率过大、数据中有 Inf、log(0)、除 0；先做梯度裁剪。</li><li>训练集精度低（欠拟合）→ 增大模型容量、训练更久、减小正则、检查数据。</li><li>训练集高、验证集低（过拟合）→ 加正则、加 Dropout、加数据增强、Early Stopping、降低模型容量。</li><li>训练集和验证集都低 → 改用更强 backbone、检查 label noise。</li><li>某些类别精度很差 → 类别不均衡，用 Focal Loss、过采样、Class-Balanced Loss。</li></ul><h2><strong>1.24 深度学习训练中是否有必要使用 L1 获得稀疏解</strong></h2><p>L1 正则化（Lasso）会使部分权重严格变为 0，得到稀疏解；L2 正则化（Ridge）使权重整体变小但不为 0。</p><p><strong>深度学习中是否需要 L1：</strong></p><ul><li>通常不需要：L2/weight decay 已经足够，且 Dropout、BN 也起到正则作用。</li><li>需要稀疏解的场景：模型剪枝（Network Pruning）、嵌入式部署、可解释性研究、特征选择。</li><li>实践中更常用的是结构化稀疏（如 group lasso、滤波器剪枝）而不是逐参数 L1。</li></ul><h2><strong>1.25 神经网络数据预处理方法有哪些？中心化/零均值，归一化</strong></h2><p><strong>零中心化（Zero-Centering）：减去均值，使数据中心在原点。</strong></p><p>x = x - μ</p><p><strong>归一化（Normalization）：缩放到单位方差或 [0,1]。</strong></p><p>Standardization:  x = (x 通道均值 [0.485, 0.456, 0.406]，除以方差 [0.229, 0.224, 0.225]。</li><li>或简单的 (x/255 - 0.5)/0.5 缩放到 [-1, 1]。</li></ul><h2><strong>1.26 推理。</li></ol><h2><strong>1.31 列举你所知道的神经网络中使用的损失函数</strong></h2><p><strong>分类任务：</strong></p><ul><li>交叉熵损失（Cross-Entropy Loss）：多分类标配。</li><li>二元交叉熵（Binary Cross-Entropy / BCE）：二分类或多标签分类。</li><li>Focal Loss：处理类别不均衡（如目标检测中前景/背景）。</li><li>Hinge Loss：SVM 风格的最大间隔分类。</li></ul><p><strong>回归任务：</strong></p><ul><li>L1 Loss / MAE：对异常值更鲁棒。</li><li>L2 Loss / MSE：处处可导，常用。</li><li>Smooth L1 / Huber Loss：L1 与 L2 的折中，Faster R-CNN 用于 bbox 回归。</li></ul><p><strong>度量学习：</strong></p><ul><li>Triplet Loss：拉近正样本，推开负样本（人脸识别）。</li><li>Contrastive Loss：对比学习的核心损失（SimCLR、MoCo）。</li><li>InfoNCE：自监督学习中广泛使用。</li></ul><p><strong>分割与生成：</strong></p><ul><li>Dice Loss / IoU Loss：医学图像分割。</li><li>Perceptual Loss：基于 VGG 高层特征比较，用于图像生成。</li><li>GAN Loss（原始 / WGAN / LSGAN）：生成对抗。</li></ul><h2><strong>1.32 什么是鞍点问题？梯度为 0，海森矩阵不定的点，不是极值点</strong></h2><p>鞍点定义：梯度 ∇L = 0，但 Hessian 矩阵 H 不定（既有正特征值又有负特征值）。在某些方向上是极小值，另一些方向上是极大值。</p><p><img src="/assets/img/dl-interview/img-06.png" /></p><p><em>图1-6  局部极小值（左，Hessian 正定）vs 鞍点（右，Hessian 不定）</em></p><p><strong>为什么鞍点在高维空间中比局部极小更常见：</strong></p><p>在 N 维空间，Hessian 是 N×N 矩阵；要它正定需要 N 个特征值都为正（概率约 (1/2)^N）。N 越大，鞍点比例呈指数级增加，局部极小反而稀少（且常常接近全局最优）。</p><p><strong>鞍点的危害：</strong></p><ul><li>梯度为 0，普通 SGD 容易停滞。</li><li>附近平坦区域（plateau）会让训练速度极慢。</li></ul><p><strong>对策：</strong></p><ul><li>Momentum / Adam 的动量项可以&quot;冲过&quot;鞍点。</li><li>mini-batch 的随机噪声本身有助于逃离鞍点。</li><li>合理初始化避开鞍点密集区。</li></ul><h2><strong>1.33 网络设计中，为什么卷积核设计尺寸都是奇数</strong></h2><ol><li>对称性 / 中心像素：奇数卷积核（3×3、5×5、7×7）有明确的几何中心，方便定位&quot;以中心像素为锚点&quot;的特征，padding 后特征图位置不偏移。</li><li>Padding 对称：奇数 kernel 用 (kernel-1)/2 的 padding 即可保持输入输出尺寸相同；偶数 kernel 需要不对称 padding，会引入位置偏移。</li><li>感受野有偏：偶数卷积核的感受野中心落在两个像素中间，存在 0.5 像素偏移。</li><li>历史惯性：经典网络（LeNet、AlexNet、VGG、ResNet）全部采用奇数核，工程实现和优化都围绕奇数核展开。</li></ol><p>例外：有些 1×1 卷积也常见，主要用于通道维变换（升降维），不涉及空间卷积。</p><h1><strong>二、卷积神经网络（CNN）</strong></h1><p>卷积神经网络是计算机视觉的基础架构。其核心思想是利用图像的空间结构特性——局部连接、权值共享、平移不变性——大幅减少参数量并提升泛化。本章覆盖 CNN 的结构、计算、经典网络（LeNet 到 DenseNet）和常见变体（1×1 卷积、空洞卷积、转置卷积等）。</p><h2><strong>2.1 卷积神经网络的结构</strong></h2><p>典型 CNN 由若干个&quot;卷积-激活-池化&quot;模块串联，最后接全连接层做分类。</p><p><strong>核心组件：</strong></p><ul><li>卷积层（Convolutional Layer）：用卷积核在输入特征图上滑动，做局部加权求和，提取空间特征。</li><li>激活函数（Activation）：通常是 ReLU，引入非线性。</li><li>池化层（Pooling Layer）：下采样，减小空间尺寸，扩大后续层的感受野，提供一定平移不变性。</li><li>全连接层（Fully-Connected, FC）：将高维特征图展平后映射到类别空间。</li><li>BN/Dropout/残差连接 等辅助组件。</li></ul><p><img src="/assets/img/dl-interview/img-07.png" /></p><p><em>图2-3  LeNet-5 的层级结构（输入 → 多次&quot;卷积+池化&quot; → 全连接 → 输出）</em></p><h2><strong>2.2 Keras 搭建 CNN</strong></h2><p>一个用 Keras 搭建经典 CNN 的简洁示例（MNIST 风格的图像分类）：</p><p>import tensorflow as tf<br />from tensorflow.keras import layers, models<br /><br />model = AvgPool → FC(120) → FC(84) → Output(10)。</li><li>激活函数用 Sigmoid/Tanh（当时还没有 ReLU），池化用平均池化。</li><li>约 6 万参数，奠定了&quot;卷积+池化+全连接&quot;的经典范式。</li></ul><h3><strong>2.2.2 AlexNet (2012, Krizhevsky et al.)</strong></h3><ul><li>深度学习&quot;复兴&quot;的标志性工作，ImageNet 2012 冠军，top-5 错误率从 26% 降到 15%。</li><li>8 层网络（5 卷积 + 3 全连接），约 6000 万参数。</li><li>首次大规模使用 ReLU 激活，解决深网梯度消失。</li><li>引入 Dropout 减轻过拟合。</li><li>在两块 GTX 580 GPU 上并行训练。</li><li>使用 LRN（局部响应归一化），后被 BN 取代。</li></ul><h3><strong>2.2.3 VGG (2014, Simonyan &amp; Zisserman)</strong></h3><ul><li>提出&quot;用堆叠的 3×3 卷积代替大卷积核&quot;的思想，结构极简但深度更大（16/19 层）。</li><li>两个堆叠的 3×3 卷积的感受野等同于一个 5×5，但参数减少 28%，且引入两次非线性。</li><li>参数主要集中在最后三层全连接（约 1.2 亿），是计算量瓶颈。</li><li>结构清晰，迁移学习效果好，至今仍作为基线模型和感知损失的特征提取器。</li></ul><h3><strong>2.2.4 Inception / GoogLeNet (2014, Szegedy et al.)</strong></h3><ul><li>提出 Inception 模块：在同一层并行使用 1×1、3×3、5×5 卷积和池化，再拼接结果。</li><li>用 1×1 卷积做&quot;瓶颈&quot;降维，大幅减少计算量。</li><li>引入辅助分类器（auxiliary head），缓解深层梯度消失，训练时贡献额外梯度。</li><li>参数仅 500 万，远小于 VGG，但 ImageNet top-5 误差更低。</li><li>后续演进：Inception-V2/V3（分解大卷积为多个小卷积，引入 BN）、V4、Inception-ResNet。</li></ul><h3><strong>2.2.5 ResNet (2015, He et al.)</strong></h3><ul><li>提出残差连接（Residual Connection），解决&quot;深网退化&quot;问题（深网在训练集上误差更高）。</li><li>核心公式：y = F(x) + x。即使 F(x) 学到的是 0，整个块仍是恒等映射。</li><li>成功训练了 152 层网络，ImageNet 2015 冠军，top-5 误差 3.57%。</li><li>彻底改变了深度学习架构设计——之后所有大模型几乎都使用跳连。</li></ul><p><img src="/assets/img/dl-interview/img-08.png" /></p><p><em>图2-4  ResNet 残差块：通过 skip connection 让梯度直通浅层</em></p><h3><strong>2.2.6 DenseNet (2017, Huang et al.)</strong></h3><ul><li>在 ResNet&quot;相加&quot;基础上更进一步：每层与前面所有层在通道维上拼接（concatenate）。</li><li>每一层都能直接访问前面所有层的特征 → 特征复用最大化。</li><li>参数效率高，相同精度下参数比 ResNet 少。</li><li>代价：显存开销大（需保留所有中间特征图用于拼接）。</li><li>引入 transition 层（Conv 1×1 + AvgPool）控制通道数膨胀。</li></ul><h2><strong>2.3 卷积层有哪些基本参数</strong></h2><ul><li>kernel_size（卷积核大小）：常见 1×1、3×3、5×5、7×7。</li><li>in_channels / out_channels：输入/输出通道数。</li><li>stride（步长）：滑窗每次移动的像素数，常见 1 或 2。</li><li>padding（填充）：在输入边缘填零，控制输出尺寸。&quot;same&quot;=输出尺寸等于输入；&quot;valid&quot;=不填零。</li><li>dilation（膨胀率/空洞率）：卷积核中元素之间的间距，用于空洞卷积。</li><li>groups（分组数）：将输入通道分组分别卷积，分组卷积 / depthwise 卷积的核心参数。</li><li>bias：是否添加偏置项（接 BN 时通常设 False）。</li></ul><h2><strong>2.4 如何计算卷积层的输出大小</strong></h2><p>对于二维卷积，输出特征图尺寸为：</p><p>H_out = ⌊ (H_in + 2·padding − dilation·(kernel − 1) − 1) / stride ⌋ + 1</p><p>W_out 同理</p><p><strong>特殊情况（dilation=1 时）：</strong></p><p>H_out = ⌊ (H_in + 2P − K) / S ⌋ + 1</p><p><strong>举例：输入 224×224，使用 kernel=7、stride=2、padding=3：</strong></p><p>H_out = (224 + 6 - 7)/2 + 1 = 112</p><p>再经过 stride=2 的最大池化：112/2 = 56。继续 stride=2 卷积 → 28、14、7，最终经 GAP 输出特征向量。这就是 ResNet 中标准的特征图尺寸序列。</p><h2><strong>2.5 如何计算卷积层参数数量</strong></h2><p>参数量 = (kernel_h × kernel_w × in_channels) × out_channels + out_channels(偏置)</p><p>其中括号内是一个卷积核的参数量，乘以 out_channels 是因为有这么多个滤波器，最后 + out_channels 是偏置（若有）。</p><p><strong>举例：3×3 卷积、in=64、out=128：</strong></p><p>3 × 3 × 64 × 128 + 128 = 73,856</p><p><strong>对比 FLOPs（每张输入的浮点运算次数）：</strong></p><p>FLOPs = H_out × W_out × kernel_h × kernel_w × in_channels × out_channels</p><h2><strong>2.6 有哪些池化方法</strong></h2><ul><li>最大池化（Max Pooling）：取窗口内最大值，保留最显著特征，是分类网络中最常用的。</li><li>平均池化（Average Pooling）：取窗口内均值，更平滑，常用于早期网络。</li><li>全局平均池化（GAP）：对每个通道整张图取平均，得到一个标量，常用于替代 FC（NIN、ResNet 后期）。</li><li>随机池化（Stochastic Pooling）：按激活值概率随机采样。</li><li>混合池化（Mixed Pooling）：max 和 average 加权组合。</li><li>LP 池化（LP Pooling）：广义 p 范数池化。</li><li>ROI Pooling / ROI Align：目标检测中将不同大小的候选框对齐到固定尺寸。</li><li>Spatial Pyramid Pooling（SPP）：多尺度池化拼接，使网络可以处理任意大小输入。</li></ul><p><img src="/assets/img/dl-interview/img-09.png" /></p><p><em>图2-2  最大池化（保留极值）vs 平均池化（保留整体趋势）</em></p><h2><strong>2.7 1×1 卷积的作用</strong></h2><ol><li>通道维度变换（升维或降维）：例如 256 通道 → 64 通道，参数量极小（256·64=16K）。</li><li>跨通道信息整合：在不改变空间尺寸的前提下，让不同通道间做线性组合。</li><li>引入非线性：1×1 卷积后接 ReLU，相当于增加一次非线性变换。</li><li>计算瓶颈结构（bottleneck）：ResNet 的 1×1→3×3→1×1 结构，先降维再升维，大幅减少 3×3 卷积的计算量。</li><li>Inception 的核心：用 1×1 卷积在多分支前做降维，使 Inception 模块计算可控。</li></ol><h2><strong>2.8 卷积层和池化层有什么区别</strong></h2><table><tr><td></td><td><p><strong>卷积层</strong></p></td><td><p><strong>池化层</strong></p></td></tr><tr><td><p>是否有参数</p></td><td><p>有可学习参数（权重 + 偏置）</p></td><td><p>无可学习参数</p></td></tr><tr><td><p>作用</p></td><td><p>提取特征</p></td><td><p>下采样、降维、提供平移不变性</p></td></tr><tr><td><p>是否改变通道数</p></td><td><p>可以改变</p></td><td><p>保持通道数不变</p></td></tr><tr><td><p>是否引入非线性</p></td><td><p>本身是线性运算</p></td><td><p>Max Pooling 是非线性的</p></td></tr><tr><td><p>反向传播</p></td><td><p>梯度按权重传递</p></td><td><p>Max 仅传给最大值位置；Avg 均分给每个位置</p></td></tr></table><h2><strong>2.9 卷积核是否一定越大越好</strong></h2><p>不是。早期 CNN 用 11×11、7×7 等大卷积核，现代网络更倾向于&quot;小卷积核+深层堆叠&quot;。</p><p><strong>大卷积核的问题：</strong></p><ul><li>参数量大：7×7 卷积参数是 3×3 的 5.4 倍。</li><li>计算量大：FLOPs 也按平方增长。</li><li>感受野虽大，但缺少层间的多次非线性变换。</li></ul><p><strong>小卷积核 + 深层堆叠的优势：</strong></p><ul><li>两个 3×3 等效于 5×5 感受野，参数少 28%。</li><li>三个 3×3 等效于 7×7 感受野，参数少 45%。</li><li>引入更多 ReLU，非线性表达能力更强。</li></ul><p>例外：最初的输入处理层有时仍用 7×7（如 ResNet 的 stem），快速扩大感受野并降低分辨率。</p><h2><strong>2.10 卷积在图像中有什么直观作用</strong></h2><p>卷积本质上是用模板（卷积核）在图像上扫描并做加权求和。不同卷积核执行不同特征提取：</p><ul><li>平滑/模糊：均值核、高斯核。</li><li>边缘检测：Sobel、Prewitt、Laplacian 核可检测水平/垂直/斜向边缘。</li><li>锐化：Laplacian 锐化核增强边缘。</li><li>浮雕、纹理提取等。</li></ul><p>深度网络中，浅层卷积核学到的就是类似 Sobel/Gabor 的边缘和纹理检测器，中层学到部件，高层学到物体级语义。</p><p><img src="/assets/img/dl-interview/img-10.png" /></p><p><em>图2-1  二维卷积运算（左：输入；中：Sobel-X 边缘检测核；右：输出特征图）</em></p><h2><strong>2.11 CNN 中空洞卷积（Dilated/Atrous Convolution）的作用是什么</strong></h2><p>空洞卷积在标准卷积核元素之间插入 (dilation-1) 个零，相当于卷积核被&quot;撑大&quot;，但参数量不变。</p><p>感受野 = (kernel - 1) × dilation + 1</p><p>例如 3×3 卷积、dilation=2 时，感受野等同于 5×5；dilation=4 时等同于 9×9。</p><p><strong>作用：</strong></p><ul><li>不增加参数和计算量的前提下，指数级扩大感受野。</li><li>保持特征图分辨率：传统下采样（池化、stride）会丢失空间信息，空洞卷积可避免。</li><li>在语义分割（DeepLab）、目标检测、语音建模（WaveNet）中广泛使用。</li></ul><p>注意：要避免&quot;网格效应&quot;（连续多次相同 dilation 会让某些像素无法被感知），常用 HDC（Hybrid Dilated Convolution，dilation 错开取值）解决。</p><h2><strong>2.12 怎样才能减少卷积层参数量</strong></h2><ol><li>使用更小的卷积核（3×3 替代 5×5/7×7）。</li><li>Bottleneck 结构：1×1 降维 → 3×3 → 1×1 升维（ResNet/Inception）。</li><li>深度可分离卷积（Depthwise Separable Conv）：将标准卷积分解为 depthwise（每通道独立）+ pointwise（1×1 跨通道），参数减少约 8~9 倍，MobileNet 核心。</li><li>分组卷积（Group Convolution）：将通道分成 G 组分别卷积，参数减为 1/G，ResNeXt 用 32 组。</li><li>1×1 卷积做通道压缩。</li><li>全局平均池化（GAP）替代 FC：VGG 的 FC 层有 1.2 亿参数，换成 GAP 后参数大幅减少。</li><li>知识蒸馏 + 量化 + 剪枝（部署阶段）。</li></ol><p><strong>深度可分离卷积参数对比：</strong></p><p>标准 3×3 卷积:        3×3×C_in×C_out</p><p>Depthwise + Pointwise: 3×3×C_in + C_in×C_out  ≈  减少 (1/C_out + 1/9) 倍</p><h2><strong>2.13 在进行卷积操作时，必须同时考虑通道和区域吗</strong></h2><p>标准卷积同时处理空间（H×W）和通道（C）信息，参数 = K×K×C_in×C_out。但这两类信息可以解耦：</p><ul><li>Depthwise 卷积：每个通道独立做空间卷积（不跨通道）。</li><li>Pointwise 卷积（1×1）：只做跨通道线性组合（不做空间聚合）。</li><li>两者结合 = Depthwise Separable Conv，分别建模空间和通道相关性。</li></ul><p>结论：不必同时考虑。空间相关性与通道相关性可以解耦建模，是轻量级网络（MobileNet、Xception）的核心思想。</p><h2><strong>2.14 采用宽卷积、窄卷积的好处有什么</strong></h2><ul><li>窄卷积（Narrow/Valid）：不做 padding，输出尺寸小于输入。优点：无边界假数据；缺点：多层后特征图迅速缩小，深层网络不友好。</li><li>宽卷积（Wide/Full）：padding=K-1，输出尺寸大于输入。可保留更多边界信息，NLP 中文本卷积常用。</li><li>Same Padding（最常用）：padding=(K-1)/2，输入输出尺寸相等，便于堆叠和残差连接。</li></ul><h2><strong>2.15 介绍反卷积（转置卷积）</strong></h2><p>反卷积（Transposed Convolution / Deconvolution，更准确的名称是&quot;转置卷积&quot;）用于上采样：将小特征图&quot;放大&quot;成大特征图，是分割（U-Net）、生成（DCGAN、Pix2Pix）的关键操作。</p><p><strong>原理：</strong></p><p>普通卷积可表示为矩阵乘法 y = C·x，其中 C 是稀疏的卷积矩阵；转置卷积就是 x = Cᵀ·y，即用 C 的转置完成&quot;由输出反推输入&quot;的形状变换（但不是真正的逆运算）。</p><p><strong>实现：</strong></p><ul><li>先在输入特征图的元素之间插零（stride 决定插零数）。</li><li>再做普通卷积。</li><li>PyTorch 中：nn.ConvTranspose2d(in, out, K, stride, padding)。</li></ul><p><strong>棋盘效应（Checkerboard Artifact）：</strong></p><p>当 stride 不能被 kernel 整除时，输出会出现规则的明暗格子。解决方法：用&quot;双线性插值上采样 + 普通卷积&quot;代替转置卷积（如 PixelShuffle、UpSample+Conv）。</p><h2><strong>2.16 如何提高卷积神经网络的泛化能力</strong></h2><ol><li>数据层面：扩充数据集、强数据增强（RandAugment、MixUp、CutMix、Mosaic）。</li><li>正则化：Weight Decay、Dropout、Stochastic Depth、DropBlock、Label Smoothing。</li><li>架构选择：合适深度和宽度、残差连接、归一化层。</li><li>训练技巧：合理学习率调度（Cosine、Warmup）、EMA 权重平均、SWA（Stochastic Weight Averaging）。</li><li>迁移学习：用大数据集（ImageNet/JFT）预训练，再在目标任务微调。</li><li>集成方法：训练多个模型投票、Snapshot Ensemble。</li><li>知识蒸馏：用大模型指导小模型。</li><li>自监督预训练：MoCo、SimCLR、MAE、DINO 等，先在无标注数据上预训练。</li></ol><h2><strong>2.17 卷积神经网络在 NLP 与 CV 领域应用的区别</strong></h2><table><tr><td></td><td><p><strong>CV (图像)</strong></p></td><td><p><strong>NLP (文本)</strong></p></td></tr><tr><td><p>输入形状</p></td><td><p>2D/3D 张量 (H,W,C)</p></td><td><p>1D 序列 (Token, Embedding_dim)</p></td></tr><tr><td><p>卷积维度</p></td><td><p>2D 卷积</p></td><td><p>1D 卷积（沿时间/序列轴）</p></td></tr><tr><td><p>卷积核形状</p></td><td><p>k×k</p></td><td><p>k×embedding_dim（覆盖整个词向量维度）</p></td></tr><tr><td><p>平移不变性</p></td><td><p>强（物体可出现在任何位置）</p></td><td><p>弱（词序有意义）</p></td></tr><tr><td><p>典型用法</p></td><td><p>特征提取、分类、检测</p></td><td><p>TextCNN 分类、字符级语言模型</p></td></tr><tr><td><p>现状</p></td><td><p>仍是 CV 主流</p></td><td><p>主流已转向 Transformer</p></td></tr></table><h2><strong>2.18 全连接、局部连接、全卷积与局部卷积的区别</strong></h2><ul><li>全连接（Fully Connected）：每个输出与每个输入都有独立权重，参数量 = N_in × N_out。</li><li>局部连接（Locally Connected）：仅与输入的局部窗口相连，但不同位置使用不同权重（不共享）。参数量介于卷积和 FC 之间。</li><li>全卷积（Fully Convolutional, FCN）：网络中没有任何全连接层，全部由卷积组成，可处理任意尺寸输入。语义分割（FCN、U-Net、DeepLab）的核心。</li><li>局部卷积（Locally Connected）= 不权值共享的卷积，适合人脸识别（不同区域学不同特征）。</li></ul><h2><strong>2.19 卷积层和全连接层的区别</strong></h2><table><tr><td></td><td><p><strong>卷积层</strong></p></td><td><p><strong>全连接层</strong></p></td></tr><tr><td><p>连接方式</p></td><td><p>局部连接 + 权值共享</p></td><td><p>全连接，无权值共享</p></td></tr><tr><td><p>参数量</p></td><td><p>小</p></td><td><p>巨大（参数瓶颈）</p></td></tr><tr><td><p>空间结构</p></td><td><p>保留空间结构（输出仍是特征图）</p></td><td><p>破坏空间结构（输入需要 Flatten）</p></td></tr><tr><td><p>输入尺寸</p></td><td><p>可变（任意 H×W）</p></td><td><p>固定（必须为预设维度）</p></td></tr><tr><td><p>典型用途</p></td><td><p>特征提取</p></td><td><p>最后分类决策（或被 GAP 替代）</p></td></tr></table><p><strong>💡 提示： </strong>一个全连接层可以等价于 1×1 卷积（输入空间大小为 1×1 时）。这一观察催生了全卷积网络（FCN），让网络可以接受任意尺寸输入。</p><h2><strong>2.20 Max pooling 如何工作？还有其他池化技术吗</strong></h2><p><strong>Max Pooling 的工作流程：</strong></p><ol><li>定义窗口大小（通常 2×2）和步长（通常等于窗口大小，无重叠）。</li><li>滑动窗口，每个位置取窗口内最大值。</li><li>输出特征图尺寸缩小（如 28×28 → 14×14）。</li></ol><p>反向传播：梯度只回传给窗口内最大值的位置，其他位置梯度为 0。</p><p>其他池化技术（详见 2.6）：Average Pooling、Global Average Pooling、Stochastic Pooling、ROI Pooling、ROI Align、SPP（空间金字塔池化）。</p><h2><strong>2.21 卷积神经网络的优点？为什么用小卷积核</strong></h2><p><strong>CNN 优点：</strong></p><ol><li>参数共享：同一卷积核扫过整张图，大幅减少参数。</li><li>局部连接：契合图像的局部相关性，提供归纳偏置。</li><li>平移不变性：池化和共享带来一定平移鲁棒。</li><li>层次化特征：浅层学边缘，中层学部件，深层学语义。</li><li>对大数据可扩展，硬件友好。</li></ol><p><strong>小卷积核的优势：</strong></p><ul><li>参数和计算量少：3×3 比 5×5 减少 28%，比 7×7 减少 81%。</li><li>堆叠多个 3×3 可达到相同感受野，但引入更多非线性。</li><li>GPU 对 3×3 卷积有专门优化（如 Winograd 算法），实测速度更快。</li></ul><h2><strong>2.22 CNN 拆成 3×1 1×3 的优点</strong></h2><p>Inception-V3 中提出的&quot;卷积核分解&quot;技巧：将 3×3 卷积拆成 3×1 + 1×3 两个非对称卷积。</p><p><strong>优点：</strong></p><ul><li>参数减少：3×3=9 个参数 vs 3+3=6 个参数，节省 33%。</li><li>计算量减少同比例。</li><li>引入一次额外的非线性激活，理论表达能力更强。</li><li>对大卷积核（5×5、7×7）拆解收益更大。</li></ul><p>限制：对小卷积核（3×3）增益有限；对感受野要求高的浅层不宜过早使用。</p><h2><strong>2.23 BN、LN、IN、GN 和 SN 已在多数任务上取代它，但 RNN 的核心思想（隐状态、序列依赖）仍是深度学习的基础概念。</p><h2><strong>3.1 RNNs 训练和传统 ANN 训练异同点</strong></h2><p><strong>相同点：</strong></p><ul><li>都使用反向传播 + 梯度下降优化。</li><li>都通过链式法则计算梯度。</li><li>都需要正则化、归一化、初始化等通用技巧。</li></ul><p><strong>不同点：</strong></p><ul><li>RNN 使用 BPTT（Backpropagation Through Time，时间反向传播）：将循环网络按时间展开成深度网络，再做标准反向传播。</li><li>参数共享：所有时间步共用同一组权重 W_xh、W_hh、W_hy。梯度需要在所有时间步上累加。</li><li>梯度消失/爆炸更严重：因为时间步可能很长（几十到几百），梯度连乘次数远多于普通深度网络。</li><li>Truncated BPTT：实际训练中通常只反向传播 K 个时间步（K 远小于序列长度），平衡效率与依赖范围。</li><li>输入是变长序列，需要 padding + mask 或 packed sequence 处理。</li></ul><p><img src="/assets/img/dl-interview/img-12.png" /></p><p><em>图3-1  RNN 按时间展开：隐状态 h_t 接收当前输入 x_t 与上一时刻 h_{t-1}</em></p><h2><strong>3.2 为什么 RNN 训练的时候 Loss 波动很大</strong></h2><ol><li>梯度爆炸/消失：长序列 BPTT 导致梯度量级在时间维度上剧烈变化，参数更新不稳定。</li><li>序列长度不同：每个 batch 内序列长度差异大，loss 量级不同。</li><li>参数共享放大噪声：同一参数在所有时间步上累加梯度，单步噪声被放大。</li><li>损失曲面陡峭：RNN 的损失曲面充满悬崖（cliff），梯度突然变大导致大步跳跃。</li></ol><p><strong>缓解方法：</strong></p><ul><li>梯度裁剪：clip_grad_norm 到 1.0~5.0。</li><li>使用 LSTM/GRU 替代标准 RNN。</li><li>降低学习率（RNN 常用 1e-3 ~ 1e-4）。</li><li>Layer Normalization 替代 Batch Normalization。</li></ul><h2><strong>3.3 RNN 中为什么会出现梯度消失</strong></h2><p>考虑标准 RNN 的隐状态更新：</p><p>h_t = tanh(W_hh · h_{t-1} + W_xh · x_t + b)</p><p>计算 ∂L/∂h_0 时，需要经过 T 次链式法则：</p><p>∂h_t / ∂h_{t-1} = diag(1 - h_t²) · W_hh</p><p>∂L / ∂h_0 = Π_{t=1..T} (∂h_t / ∂h_{t-1}) ≈ (W_hh)^T · tanh'(·)^T</p><p>如果 W_hh 的最大特征值 &lt; 1，梯度在时间维呈指数衰减（消失）；&gt; 1 则指数增长（爆炸）。</p><p>而 tanh'(·) 最大为 1（在 0 处），饱和区接近 0，进一步加剧梯度消失。</p><h2><strong>3.4 如何解决 RNN 中的梯度消失问题</strong></h2><ol><li>使用 LSTM / GRU：门控机制为梯度提供&quot;高速通道&quot;（cell state），不经过反复的 tanh 衰减。</li><li>合理初始化：使用单位矩阵初始化 W_hh（IRNN）或正交初始化（让 ‖W_hh‖₂ ≈ 1）。</li><li>梯度裁剪：主要解决爆炸；消失需要架构改进。</li><li>使用 ReLU 替代 tanh（但需配合正交初始化或单位矩阵初始化，否则容易爆炸）。</li><li>Layer Normalization：让每个时间步的激活分布稳定。</li><li>残差连接：跨时间步的 skip connection。</li><li>从根本上：用 Transformer / Self-Attention 替代 RNN，路径长度从 O(T) 降到 O(1)。</li></ol><h2><strong>3.5 CNN VS RNN</strong></h2><table><tr><td></td><td><p><strong>CNN</strong></p></td><td><p><strong>RNN</strong></p></td></tr><tr><td><p>处理数据</p></td><td><p>图像、空间网格</p></td><td><p>序列、时间数据</p></td></tr><tr><td><p>连接方式</p></td><td><p>局部连接 + 权值共享</p></td><td><p>时间循环 + 权值共享</p></td></tr><tr><td><p>关键归纳偏置</p></td><td><p>平移不变性</p></td><td><p>时间不变性</p></td></tr><tr><td><p>并行性</p></td><td><p>高（每层可并行）</p></td><td><p>低（必须按时间顺序）</p></td></tr><tr><td><p>长程依赖</p></td><td><p>需深层堆叠或空洞卷积</p></td><td><p>LSTM/GRU 支持，但 &gt;100 步仍困难</p></td></tr><tr><td><p>训练稳定性</p></td><td><p>高</p></td><td><p>较低（梯度消失/爆炸）</p></td></tr><tr><td><p>现状</p></td><td><p>CV 主力，部分被 ViT 替代</p></td><td><p>基本被 Transformer 替代</p></td></tr></table><h2><strong>3.6 Keras 搭建 RNN</strong></h2><p>文本分类的 RNN 示例：</p><p>import tensorflow as tf<br />from tensorflow.keras import layers, models<br /><br />vocab_size = 10000<br />embed_dim = 128<br />max_len = 100<br /><br />model = models.Sequential([<br />    layers.Embedding(vocab_size, embed_dim, input_length=max_len),<br />    layers.Bidirectional(layers.LSTM(64, epochs=10, batch_size=64,<br />          validation_data=(x_val, y_val))</p><h1><strong>四、长短期记忆网络（LSTM）</strong></h1><p>LSTM 由 Hochreiter &amp; Schmidhuber 于 1997 年提出，通过精心设计的&quot;门控机制&quot;解决了标准 RNN 的梯度消失问题，成为 2014~2018 年间 NLP/语音领域的主力架构。</p><h2><strong>4.1 LSTM 结构推导，为什么比 RNN 好</strong></h2><p>LSTM 引入两条信息流：cell state C（长期记忆）和 hidden state h（短期记忆），通过三个门控制信息的读写擦除。</p><p><img src="/assets/img/dl-interview/img-13.png" /></p><p><em>图4-1  LSTM 单元内部结构：遗忘门、输入门、输出门 + 细胞状态 C</em></p><p><strong>完整公式（输入 x_t，前一状态 h_{t-1}, C_{t-1}）：</strong></p><p>f_t = σ(W_f · [h_{t-1}, x_t] + b_f)                   遗忘门</p><p>i_t = σ(W_i · [h_{t-1}, x_t] + b_i)                   输入门</p><p>~C_t = tanh(W_C · [h_{t-1}, x_t] + b_C)               候选细胞状态</p><p>C_t = f_t ⊙ C_{t-1} + i_t ⊙ ~C_t                      更新细胞状态</p><p>o_t = σ(W_o · [h_{t-1}, x_t] + b_o)                   输出门</p><p>h_t = o_t ⊙ tanh(C_t)                                 隐状态输出</p><p>其中 σ 是 Sigmoid，输出 [0,1] 起&quot;门&quot;的作用；⊙ 是逐元素相乘。</p><p><strong>为什么比 RNN 好：</strong></p><ol><li>梯度高速通道：cell state 的更新 C_t = f_t ⊙ C_{t-1} + i_t ⊙ ~C_t 是加法形式（而非 RNN 的 tanh(W·h)），梯度可直接沿 cell state 传递，不会指数衰减。</li><li>门控可选择性遗忘 / 记忆：模型自适应决定保留什么、忘掉什么。</li><li>对长程依赖更友好：理论上可处理 100+ 时间步的依赖。</li><li>训练更稳定：实践中 loss 曲线比 RNN 平滑得多。</li></ol><h2><strong>4.2 为什么 LSTM 模型中既存在 sigmoid 又存在 tanh 两种激活函数，而不是选择统一一种 sigmoid 或者 tanh</strong></h2><p>两者在 LSTM 中扮演不同角色，不能互换：</p><p><strong>Sigmoid 用于门（gate）：</strong></p><ul><li>输出范围 [0, 1]，恰好对应&quot;完全保留(1) / 完全遗忘(0) / 部分通过(0.5)&quot;的开关语义。</li><li>用于 f_t, i_t, o_t 三个门。</li></ul><p><strong>Tanh 用于状态值：</strong></p><ul><li>输出范围 [-1, 1]，零均值，使 cell state 可以正向或负向编码信息，且数值范围有界（避免无限增长）。</li><li>用于候选 ~C_t 和 h_t 的输出激活。</li></ul><p>如果都用 sigmoid，状态值无法为负，表达能力受限；如果都用 tanh，门无法表达&quot;完全关闭&quot;（tanh(0)=0 但 tanh 不能在饱和区给出 0~1 的概率语义）。</p><h2><strong>4.3 LSTM 中为什么经常是两层双向 LSTM</strong></h2><p>双向 LSTM（Bi-LSTM）同时使用前向和后向两个 LSTM，将两者的隐状态拼接：</p><p>h_t = [→h_t ; ←h_t]</p><p><strong>优点：</strong></p><ul><li>上下文双向感知：当前 token 的表示同时融合了过去和未来信息（对句子级 NLU 至关重要）。</li><li>提升表达能力：单向只能从左到右建模，双向可捕捉双向语义依赖。</li></ul><p><strong>为什么用两层：</strong></p><ul><li>单层 Bi-LSTM 表达能力有限；两层能学到更高级的语义特征。</li><li>层数过多（3+）训练不稳定且收益递减。</li><li>两层是性能/效率/稳定性的常见折中。</li></ul><p><strong>⚠️ 注意： </strong>注意：Bi-LSTM 不适用于&quot;在线/流式&quot;场景（如实时语音识别），因为需要看到完整序列。这种情况下只能用单向。</p><h2><strong>4.4 RNN 扩展改进</strong></h2><h3><strong>4.4.1 Bidirectional RNNs</strong></h3><p>如上文所述，由两个方向相反的 RNN 组成，输出拼接两者的隐状态。常见于命名实体识别、机器翻译编码器等。</p><h3><strong>4.4.2 CNN-LSTMs</strong></h3><p>先用 CNN 提取局部特征，再用 LSTM 建模时序。典型应用：</p><ul><li>视频分类：CNN 抽取每帧特征，LSTM 建模帧间时序。</li><li>OCR：CNN 提取图像特征，LSTM/CTC 解码为文字。</li><li>语音识别：CNN 提取声学特征，LSTM 建模发音序列。</li></ul><h3><strong>4.4.3 Bidirectional LSTMs</strong></h3><p>Bi-LSTM 是 Bi-RNN 用 LSTM 替换标准 RNN 的版本，是 NLP 中长期的事实标准（被 BERT/Transformer 替代之前）。</p><h3><strong>4.4.4 GRU（Gated Recurrent Unit）</strong></h3><p>Cho 等人于 2014 年提出，是 LSTM 的简化版：将 LSTM 的&quot;输入门 + 遗忘门&quot;合并为&quot;更新门&quot;，去掉单独的 cell state。</p><p>z_t = σ(W_z · [h_{t-1}, x_t])              更新门</p><p>r_t = σ(W_r · [h_{t-1}, x_t])              重置门</p><p>~h_t = tanh(W · [r_t ⊙ h_{t-1}, x_t])      候选状态</p><p>h_t = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ ~h_t    最终输出</p><p>优点：参数比 LSTM 少 ~25%，训练快；缺点：表达能力略弱，但在大多数任务上与 LSTM 性能相当。</p><h2><strong>4.5 LSTM、RNN、GRU 区别</strong></h2><table><tr><td></td><td><p><strong>RNN</strong></p></td><td><p><strong>LSTM</strong></p></td><td><p><strong>GRU</strong></p></td></tr><tr><td><p>门数量</p></td><td><p>0</p></td><td><p>3（输入/遗忘/输出）</p></td><td><p>2（更新/重置）</p></td></tr><tr><td><p>状态</p></td><td><p>隐状态 h</p></td><td><p>隐状态 h + 细胞状态 C</p></td><td><p>隐状态 h</p></td></tr><tr><td><p>参数量</p></td><td><p>最少</p></td><td><p>最多（约 4 倍 RNN）</p></td><td><p>中等（约 3 倍 RNN）</p></td></tr><tr><td><p>长程依赖</p></td><td><p>差（梯度消失）</p></td><td><p>强</p></td><td><p>强（与 LSTM 接近）</p></td></tr><tr><td><p>训练稳定性</p></td><td><p>低</p></td><td><p>高</p></td><td><p>高</p></td></tr><tr><td><p>训练速度</p></td><td><p>快</p></td><td><p>慢</p></td><td><p>中等</p></td></tr><tr><td><p>典型场景</p></td><td><p>短序列、教学</p></td><td><p>长序列、NLP/语音</p></td><td><p>NLP、轻量场景</p></td></tr></table><h2><strong>4.6 LSTM 是如何实现长短期记忆功能的</strong></h2><ol><li>细胞状态 C 作为长期记忆载体：以加法方式更新，梯度沿 C 直接传递，不会消失。</li><li>遗忘门 f_t：决定保留多少旧记忆（C_{t-1}）。当 f_t≈1 时，长期记忆可以传播极远；当 f_t≈0 时，主动遗忘。</li><li>输入门 i_t：决定将多少新信息写入 C。</li><li>输出门 o_t：决定将 C 中多少信息作为短期输出 h_t 暴露给下游层。</li></ol><p>因此，&quot;长期&quot;由 cell state 承担（缓慢更新，可保存上百时间步前的信息）；&quot;短期&quot;由 hidden state 承担（每步都重新计算）。</p><h2><strong>4.7 LSTM 的原理、写 LSTM 的公式、手推 LSTM 的梯度反向传播</strong></h2><p>原理与公式见 4.1。下面手推关键梯度：</p><p><strong>设损失函数为 L，已知 ∂L/∂h_t 和 ∂L/∂C_t。</strong></p><p><strong>Step 1：计算对各门的梯度</strong></p><p>∂L/∂o_t = (∂L/∂h_t) ⊙ tanh(C_t)</p><p>∂L/∂C_t += (∂L/∂h_t) ⊙ o_t ⊙ (1 - tanh²(C_t))</p><p>∂L/∂f_t = (∂L/∂C_t) ⊙ C_{t-1}</p><p>∂L/∂i_t = (∂L/∂C_t) ⊙ ~C_t</p><p>∂L/∂~C_t = (∂L/∂C_t) ⊙ i_t</p><p><strong>Step 2：通过激活函数</strong></p><p>∂L/∂a_o = ∂L/∂o_t ⊙ o_t ⊙ (1 - o_t)        (σ'(x) = σ(x)(1-σ(x)))</p><p>∂L/∂a_f = ∂L/∂f_t ⊙ f_t ⊙ (1 - f_t)</p><p>∂L/∂a_i = ∂L/∂i_t ⊙ i_t ⊙ (1 - i_t)</p><p>∂L/∂a_C = ∂L/∂~C_t ⊙ (1 - ~C_t²)             (tanh'(x) = 1-tanh²)</p><p><strong>Step 3：累积权重梯度（4 个矩阵均按时间累加）</strong></p><p>∂L/∂W_f += ∂L/∂a_f · [h_{t-1}, x_t]ᵀ</p><p>其他 W_i、W_C、W_o 类似。</p><p><strong>Step 4：向前一时间步传递</strong></p><p>∂L/∂C_{t-1} = (∂L/∂C_t) ⊙ f_t                关键：梯度乘以 f_t 而不是 tanh 导数</p><p>∂L/∂h_{t-1} = (Σ_gate ∂L/∂a_gate · W_gate)[h 部分]</p><p><strong>🔑 要点： </strong>关键观察：cell state 的梯度只乘以 f_t（一个 sigmoid 输出，可控），不像普通 RNN 要乘 tanh 导数 + W_hh 的特征值。这就是 LSTM 缓解梯度消失的本质原因。</p><h1><strong>五、反向传播（Backpropagation）</strong></h1><p>反向传播是神经网络训练的核心算法，由 Rumelhart、Hinton 和 Williams 于 1986 年系统化提出。其本质是利用链式法则，将损失函数对参数的梯度从输出层逐层回传到每个参数，从而支持梯度下降优化。</p><h2><strong>5.1 什么是反向传播</strong></h2><p>反向传播（Backpropagation, BP）是一种高效计算神经网络中所有参数对损失函数梯度的算法。它基于微积分中的链式法则，从输出层开始，按计算图逆向逐层计算梯度，时间复杂度与前向传播相当（O(参数量)）。</p><p><img src="/assets/img/dl-interview/img-14.png" /></p><p><em>图5-1  反向传播示意：蓝色为前向计算，红色虚线为梯度逆向传播</em></p><p><strong>关键概念：</strong></p><ul><li>计算图（Computational Graph）：将复合函数表示为节点和边，每个节点是一个原子运算。</li><li>链式法则：∂L/∂x = (∂L/∂y) · (∂y/∂x)，把整体梯度分解为局部梯度的乘积。</li><li>自动微分（Autograd）：PyTorch、TensorFlow 通过记录计算图自动实现反向传播，无需手写。</li></ul><h2><strong>5.2 反向传播是如何工作的</strong></h2><p>以三层 MLP 为例，演示完整流程：</p><p>网络：x → [W₁, b₁] → a₁=σ(z₁) → [W₂, b₂] → a₂=σ(z₂) → [W₃, b₃] → ŷ → L(ŷ, y)</p><p><strong>前向传播：</strong></p><p>z₁ = W₁·x + b₁,   a₁ = σ(z₁)</p><p>z₂ = W₂·a₁ + b₂, a₂ = σ(z₂)</p><p>ŷ = W₃·a₂ + b₃,   L = (1/2)(ŷ - y)²</p><p><strong>反向传播（从右往左，逐层应用链式法则）：</strong></p><p>δ₃ = ∂L/∂ŷ = ŷ - y</p><p>∂L/∂W₃ = δ₃ · a₂ᵀ      ∂L/∂b₃ = δ₃</p><p>δ₂ = (W₃ᵀ · δ₃) ⊙ σ'(z₂)</p><p>∂L/∂W₂ = δ₂ · a₁ᵀ      ∂L/∂b₂ = δ₂</p><p>δ₁ = (W₂ᵀ · δ₂) ⊙ σ'(z₁)</p><p>∂L/∂W₁ = δ₁ · xᵀ        ∂L/∂b₁ = δ₁</p><p>一般规律：每层的局部梯度 δ_l = (上一层 δ_{l+1} 通过权重 W_{l+1}ᵀ 反传) ⊙ 本层激活的导数。该层权重梯度 = δ_l 与输入的外积。</p><h2><strong>5.3 为什么需要反向传播</strong></h2><ol><li>计算效率：要训练神经网络就要计算损失对所有参数的梯度。若用数值微分（有限差分），每个参数都需要一次完整前向计算，复杂度 O(参数量²)。反向传播仅需一次反向计算（与前向相当），复杂度 O(参数量)，是唯一可行的算法。</li><li>精确性：数值微分有舍入误差和步长选择问题；反向传播（解析微分）给出精确梯度（除浮点误差外）。</li><li>通用性：任何由可微原子运算构成的计算图都能自动微分，无需对每个网络重新推导。</li><li>支持深度学习：没有 BP，训练 10 层以上的网络就不可行。</li></ol><h2><strong>5.4 手推 BP</strong></h2><p>考虑最简单的二层网络做二分类，含一个隐层（ReLU）和 sigmoid 输出。批量大小 = 1，输入 x ∈ R^d，标签 y ∈ {0,1}。</p><p><strong>前向传播：</strong></p><p>z₁ = W₁·x + b₁                   (W₁ ∈ R^{h×d}, b₁ ∈ R^h)</p><p>a₁ = ReLU(z₁)                    (∈ R^h)</p><p>z₂ = w₂·a₁ + b₂                  (w₂ ∈ R^h, b₂ ∈ R)</p><p>ŷ = σ(z₂) = 1/(1+e^{-z₂})         (∈ (0,1))</p><p>L = -[y·log(ŷ) + (1-y)·log(1-ŷ)]  (BCE 损失)</p><p><strong>反向传播：</strong></p><p><strong>① 输出层梯度（Sigmoid + BCE 的优雅性质）：</strong></p><p>∂L/∂z₂ = ŷ - y           （BCE+Sigmoid 抵消，得到非常简洁的表达）</p><p><strong>② 输出层参数梯度：</strong></p><p>∂L/∂w₂ = (ŷ - y) · a₁ᵀ</p><p>∂L/∂b₂ = ŷ - y</p><p><strong>③ 传到隐层：</strong></p><p>∂L/∂a₁ = (ŷ - y) · w₂</p><p>∂L/∂z₁ = ∂L/∂a₁ ⊙ ReLU'(z₁) = ∂L/∂a₁ ⊙ 𝟙(z₁ &gt; 0)</p><p><strong>④ 隐层参数梯度：</strong></p><p>∂L/∂W₁ = ∂L/∂z₁ · xᵀ      (外积，得到 h×d 矩阵)</p><p>∂L/∂b₁ = ∂L/∂z₁</p><p><strong>参数更新：</strong></p><p>W₁ ← W₁ - lr · ∂L/∂W₁,    b₁ ← b₁ - lr · ∂L/∂b₁</p><p>w₂ ← w₂ - lr · ∂L/∂w₂,    b₂ ← b₂ - lr · ∂L/∂b₂</p><p><strong>🔑 要点： </strong>面试常考：为什么交叉熵 + Sigmoid 比 MSE + Sigmoid 好？因为前者的 ∂L/∂z = ŷ-y 不含 σ'(z) 项，避免了饱和区梯度消失；后者的 ∂L/∂z = (ŷ-y)·ŷ·(1-ŷ) 在 ŷ 接近 0 或 1 时梯度极小。</p><h1><strong>第六章 生成对抗网络（GAN）</strong></h1><p>生成对抗网络（Generative Adversarial Network, GAN）由 Goodfellow 等人在 2014 年提出，是一类通过&quot;对抗博弈&quot;实现数据分布建模的生成模型。它包含两个相互对抗的网络：生成器（Generator, G）负责&quot;造假&quot;，把随机噪声 z 映射为接近真实分布的样本 G(z)；判别器（Discriminator, D）负责&quot;辨真假&quot;，输出输入样本来自真实分布的概率 D(x)。两者通过 minimax 博弈共同优化：</p><p>min_G max_D V(D, G) = E_{x~p_data}[log D(x)] + E_{z~p_z}[log(1 - D(G(z)))]</p><p>理论上，当博弈达到全局最优时，G 学到了真实数据分布 p_data，D(x) 在任意点恒等于 1/2（无法区分真假）。但实际训练充满陷阱：模式崩溃、梯度消失、训练不稳定，由此衍生出 DCGAN、WGAN、StyleGAN 等一系列变体。</p><p><img src="/assets/img/dl-interview/img-15.png" /></p><p><em>图6-1 GAN整体架构：G 和 D 的对抗博弈</em></p><h2><strong>6.1 生成器（Generator）</strong></h2><p><strong>【作用】</strong></p><p>生成器是一个可微分函数 G: Z → X，把低维的隐变量空间 Z（通常是各向同性高斯 N(0, I)）映射到高维数据空间 X（图像 / 音频 / 文本等）。它的目标是让生成样本 G(z) 的分布 p_g 尽量逼近真实数据分布 p_data。</p><p><strong>【典型结构（以 DCGAN 为例）】</strong></p><p>DCGAN（Deep Convolutional GAN）确立了卷积生成器的事实标准：输入 100 维隐向量 z，通过一个全连接层 reshape 为 4×4×1024 的张量，然后通过若干层&quot;转置卷积（Deconvolution / Transposed Convolution）&quot;逐步上采样，最终输出 64×64×3 的彩色图像。</p><p><img src="/assets/img/dl-interview/img-16.png" /></p><p><em>图6-2 DCGAN 生成器：噪声 → 图像的逐步上采样</em></p><p><strong>【关键设计原则】</strong></p><ul><li>用转置卷积或 PixelShuffle 替代池化层，实现可学习的上采样。</li><li>每个隐藏层后接 BatchNorm + ReLU，加速收敛并稳定训练。</li><li>输出层使用 tanh 激活，把像素值映射到 [-1, 1]（与真实图像预处理一致）。</li><li>避免在生成器中使用 dropout（会引入额外噪声，干扰生成质量）。</li></ul><p><strong>【上采样的三种主流方式】</strong></p><table><tr><td><p><strong>方法</strong></p></td><td><p><strong>原理</strong></p></td><td><p><strong>优点</strong></p></td><td><p><strong>缺点</strong></p></td></tr><tr><td><p>转置卷积</p></td><td><p>通过插零+卷积实现可学习上采样</p></td><td><p>端到端可训练；表达力强</p></td><td><p>容易产生&quot;棋盘格伪影&quot;</p></td></tr><tr><td><p>Upsample+Conv</p></td><td><p>先用 nearest/bilinear 上采样，再用普通卷积</p></td><td><p>伪影少；实现简单</p></td><td><p>可学习参数少</p></td></tr><tr><td><p>PixelShuffle</p></td><td><p>把通道维 r²·C reshape 为空间维 (rH)×(rW)</p></td><td><p>无伪影；ESRGAN 等超分常用</p></td><td><p>需配套设计输入通道数</p></td></tr></table><p><strong>📌 备注： </strong>&quot;棋盘格伪影&quot;产生的原因：当转置卷积的 kernel_size 不能被 stride 整除时，不同输出位置被卷积核覆盖的次数不均匀，造成周期性的亮度差。一个简单的修复：让 kernel_size 是 stride 的整数倍（如 4×4 stride 2），或者换用 Upsample+Conv。</p><p><strong>【生成器的进阶变体】</strong></p><ul><li>Progressive GAN（NVIDIA, 2017）：从 4×4 分辨率开始，逐步加层升到 1024×1024，显著提升高分辨率生成质量。</li><li>StyleGAN / StyleGAN2 / StyleGAN3：把隐向量 z 通过 mapping network 映射为 style 向量 w，通过 AdaIN（自适应实例归一化）在各分辨率层注入风格，实现属性解耦与精细控制。</li><li>BigGAN（2018）：超大 batch + 大模型 + 类别条件，在 ImageNet 上 256×256 生成达到 SOTA。</li><li>扩散模型（Diffusion，2020 起）：用 U-Net 去噪迭代生成，已在很多场景超越 GAN（如 Stable Diffusion）。</li></ul><h2><strong>6.2 判别器（Discriminator）</strong></h2><p><strong>【作用】</strong></p><p>判别器本质上是一个二分类器 D: X → [0, 1]，输入一张图像，输出它来自真实数据集而非生成器的概率。它给 G 提供梯度信号：如果 D 能轻易区分真假，说明 G 还不够好；如果 D 也搞不清，说明 G 学得不错。</p><p><strong>【典型结构（DCGAN 判别器）】</strong></p><p>判别器结构基本是生成器的&quot;镜像&quot;——通过若干层 stride-2 卷积逐步下采样，最后 flatten 接全连接和 sigmoid。</p><p><img src="/assets/img/dl-interview/img-17.png" /></p><p><em>图6-3 DCGAN 判别器：图像 → 真假概率的逐步下采样</em></p><p><strong>【关键设计原则】</strong></p><ul><li>用 stride-2 卷积替代池化，让下采样过程可学习。</li><li>使用 LeakyReLU（α=0.2）而不是 ReLU——LeakyReLU 在负区间有小斜率，能让 D 给 G 提供更稳定的梯度信号，避免 ReLU&quot;死神经元&quot;在对抗训练中放大问题。</li><li>第一层（输入层）一般不使用 BatchNorm，避免破坏输入分布；其余层均使用 BN（WGAN-GP 则用 LayerNorm）。</li><li>全卷积结构（无 padding 信息泄漏）有利于支持任意分辨率输入。</li></ul><p><strong>【判别器的进阶设计】</strong></p><ul><li>PatchGAN（pix2pix）：D 不输出全局的一个数，而是输出 N×N 的&quot;patch 真假图&quot;，每个像素负责判断输入图中对应感受野（如 70×70）的局部 patch。这强迫 G 在细节纹理上更逼真，对图像翻译任务很有效。</li><li>Spectral Normalization 谱归一化（SN-GAN, 2018）：把 D 中每个卷积/全连接层的权重矩阵除以它的最大奇异值，保证 D 是 1-Lipschitz 函数。这是稳定 GAN 训练的&quot;杀手锏&quot;，几乎成为标配。</li><li>Self-Attention（SAGAN）：在中间分辨率层引入自注意力，让 D 能捕获长程依赖（如物体的整体结构）。</li><li>Projection Discriminator（cGAN-Projection）：在条件 GAN 中，把类别 embedding 通过内积与中间特征相乘，比把 label concat 到 D 输入效果更好。</li></ul><p><strong>⚠️ 注意： </strong>一个常见误区：D 太强会导致 G 训练崩溃。当 D 很容易达到 100% 准确时，log(1 - D(G(z))) 接近 0，G 拿到的梯度趋近 0。解决方案：① 用非饱和损失 -log D(G(z)) 替代 log(1 - D(G(z)))；② 限制 D 的容量（更少层 / 谱归一化）；③ 用 Wasserstein 距离替换 JS 散度。</p><h2><strong>6.3 GAN 训练技巧</strong></h2><p>GAN 训练以&quot;难以稳定&quot;著称——它不是一个普通的优化问题，而是 minimax 博弈，损失函数曲线和模型性能的关联很弱。下面是工业界与学术界长期沉淀的实用技巧。</p><h3><strong>6.3.1 缓解模式崩溃（Mode Collapse）</strong></h3><p><strong>【现象】</strong></p><p>模式崩溃指生成器只生成数据分布中的少数几种模式（如人脸 GAN 只会生成一两张脸，反复出现）。原因是 G 找到了一个能&quot;骗过&quot; D 的&quot;局部最优解&quot;，于是停止探索其他模式。</p><p><img src="/assets/img/dl-interview/img-18.png" /></p><p><em>图6-4 模式崩溃：G 仅覆盖目标分布的少数几个模式</em></p><p><strong>【缓解方法】</strong></p><ul><li>Minibatch Discrimination：在 D 中引入一个层，让 D 能&quot;看到&quot;同一 batch 中其他样本的统计信息，从而惩罚 G 生成相互雷同的样本。</li><li>Unrolled GAN：训练 G 时，让它考虑 D 未来 K 步的更新，避免被 D 的瞬时弱点利用。</li><li>PacGAN：把 D 的输入改成 m 个样本的串联，从而让 D 能感知样本多样性。</li><li>WGAN：换用 Wasserstein 距离作为目标，从根本上改善梯度信号（见 6.3.3）。</li></ul><h3><strong>6.3.2 谱归一化与 1-Lipschitz 约束</strong></h3><p>现代 GAN 训练几乎默认使用谱归一化（Spectral Normalization, SN）：把每层权重矩阵 W 替换为 W / σ(W)，其中 σ(W) 是 W 的最大奇异值（用幂迭代估计，几乎不增加计算）。它使 D 的 Lipschitz 常数被约束在 1，从而稳定训练。</p><p># PyTorch 一行启用谱归一化<br />import torch.nn.utils as utils<br />self.conv = utils.spectral_norm(nn.Conv2d(in_c, out_c, 4, 2, 1))</p><h3><strong>6.3.3 Wasserstein 距离与梯度惩罚（WGAN-GP）</strong></h3><p><strong>【为什么要用 Wasserstein 距离】</strong></p><p>原始 GAN 用 JS 散度衡量 p_g 和 p_data 的差异，但当两个分布的&quot;支撑集&quot;不重叠时（高维分布大概率如此），JS 散度恒为 log2，梯度几乎为 0——这就是 GAN 训练初期&quot;D 太强 G 学不动&quot;的根源。</p><p>Wasserstein 距离（推土机距离）衡量&quot;把一个分布搬成另一个分布需要的最小代价&quot;，即使两个分布无重叠，它也能给出一个有意义的距离值，因此提供平滑的梯度。</p><p><img src="/assets/img/dl-interview/img-19.png" /></p><p><em>图6-5 JS 散度（不重叠时无梯度） vs Wasserstein 距离（处处可导）</em></p><p><strong>【WGAN 损失】</strong></p><p>L_D = E_{x~p_data}[D(x)] - E_{z~p_z}[D(G(z))]</p><p>L_G = -E_{z~p_z}[D(G(z))]</p><p>其中 D 不再输出概率（无 sigmoid），而是一个标量 critic（评分器）。为了让 Wasserstein 距离的对偶形式成立，D 必须是 1-Lipschitz 的。原始 WGAN 用&quot;权重裁剪&quot;实现（粗暴且效果差），WGAN-GP 改用&quot;梯度惩罚&quot;：</p><p>L_D += λ · E_{x̂}[(‖∇_{x̂} D(x̂)‖_2 - 1)²]</p><p>其中 x̂ 是真实样本 x 和生成样本 G(z) 的随机线性插值，λ 通常取 10。WGAN-GP 是目前最稳健的 GAN 训练方案之一。</p><h3><strong>6.3.4 双时间尺度更新规则（TTUR）</strong></h3><p>传统做法是 G 和 D 使用相同的学习率，并且每 step D 多更新 k 次（如 k=5）。TTUR（Two Time-scale Update Rule）证明：让 D 的学习率比 G 大（如 D=4e-4, G=1e-4），两者均每 step 更新 1 次，能更快收敛到纳什均衡，并提升 FID。这是 BigGAN、StyleGAN 等大模型的标准做法。</p><h3><strong>6.3.5 标签平滑与输入噪声</strong></h3><ul><li>One-sided label smoothing：把&quot;真实样本&quot;的标签从 1.0 改为 0.9（不要改 fake 的 0.0），防止 D 输出过于自信，对应缓解 G 的梯度消失。</li><li>给 D 的输入加上高斯噪声（instance noise）：在训练初期给真实图像和生成图像都加上随机噪声，强制两个分布有重叠，从而让 D 提供有效梯度；随训练进行逐渐衰减噪声方差。</li></ul><h3><strong>6.3.6 特征匹配与 Perceptual Loss</strong></h3><p>改变 G 的目标：不再仅让 D(G(z)) 接近 1，而是让 G(z) 在 D 中间层的特征激活均值接近真实样本：</p><p>L_FM = ‖E_x[f(x)] - E_z[f(G(z))]‖²₂</p><p>这种方式比单纯优化 D 的最终输出更稳定，特别适合半监督 GAN。另一相关技巧是 Perceptual Loss：使用预训练 VGG 的中间层作为&quot;语义距离&quot;度量，常用于图像翻译和超分辨率。</p><h3><strong>6.3.7 模型表达能力的释放</strong></p><p>同一个网络结构，超参数不同会让模型处于完全不同的工作点：欠拟合（学习率太小、正则太强）、过拟合（学习率合适但 epoch 太多、正则太弱）、不收敛（学习率过大）、收敛慢（batch 太大、没有 warmup）。调优本质上是把模型推到&quot;恰好饱和但未过拟合&quot;的最佳工作点。</p><p><strong>② 数据集和任务差异</strong></p><p>在 ImageNet 上 SOTA 的超参数（如 lr=0.1, bs=256, SGD+momentum）直接用到自己的 5000 张图像数据集上，大概率会发散。数据规模、类别数、噪声水平、长尾程度都会影响最佳超参数。</p><p><strong>③ 计算资源约束</strong></p><p>工业部署常面临&quot;用更小的模型 / 更少的训练时间达到同样精度&quot;的压力。超参数调优能在不改架构的前提下，把训练时间缩短 2~5 倍（如启用 OneCycleLR + LARS + 大 batch）。</p><h2><strong>7.3 bs=8192~32768，用于加速训练）。两者的应对策略截然不同。</p><h3><strong>7.4.1 batch 极小（显存受限）</strong></h3><p><strong>【主要问题】</strong></p><ul><li>梯度噪声大，训练不稳定。</li><li>BatchNorm 统计量估计不准（bs&lt;8 时尤其严重），导致训练-测试分布不匹配。</li><li>每个 step 利用率低，整体训练时间长。</li></ul><p><strong>【解决方案】</strong></p><ul><li>梯度累积（Gradient Accumulation）：累积 N 个小 batch 的梯度再更新一次参数，等效大 batch 训练。代码上只需要 loss.backward() 多次后再 optimizer.step()。</li><li>使用 GroupNorm 或 LayerNorm 替代 BatchNorm。GN 对 batch size 不敏感，在检测/分割中（bs 常为 1~4）几乎总是优于 BN。</li><li>使用 SyncBN（同步 BN）：在多卡场景下把所有卡上的 batch 统计量合并计算，等效增大 BN 的有效 batch。</li><li>混合精度训练（AMP / fp16）：节省一半显存，从而把 batch 翻倍。</li><li>梯度检查点（Gradient Checkpointing）：用计算换显存，能把 batch 提高 2~4 倍。</li></ul><h3><strong>7.4.2 batch 极大（分布式加速）</strong></h3><p><strong>【主要问题】</strong></p><ul><li>大 batch 减少梯度噪声，泛化能力下降（落入&quot;sharp minima&quot;）。</li><li>达到相同精度需要的 epoch 数显著增加。</li><li>训练初期容易发散。</li></ul><p><strong>【解决方案】</strong></p><ul><li>线性 LR 缩放规则（Goyal et al. 2017, Facebook）：bs 扩大 k 倍，lr 也扩大 k 倍。该规则在 bs ≤ 8K 内有效。</li><li>LR Warmup：训练初期前 5~10 个 epoch 从 lr/k 线性升到目标 lr，避免大 lr 一上来就发散。</li><li>LARS / LAMB 优化器：对每层权重单独计算&quot;信任比&quot;（trust ratio），缓解大 batch 下不同层学习率不匹配的问题。LARS 用于 SGD 系列（ResNet 等），LAMB 用于 Adam 系列（BERT 等）。BERT 训练把时间从 3 天压缩到 76 分钟，靠的就是 LAMB + bs=32K。</li><li>数据并行 + 梯度同步（All-Reduce）：用 NCCL/Horovod/PyTorch DDP 实现。注意大 batch 下的&quot;learning rate cooldown&quot;也很关键。</li></ul><table><tr><td><p><strong>场景</strong></p></td><td><p><strong>batch vision 和 NLP 任务的标准流程都是&quot;加载预训练权重 → 在下游任务上微调&quot;。预训练模型能利用大规模数据（ImageNet、LAION、Wikipedia 等）学到的通用特征，让下游任务用更少数据、更短时间达到更高精度。本节系统讨论微调的方方面面。</p><h3><strong>7.5.1 什么是微调（fine-tune）？</strong></h3><p>微调是迁移学习的一种典型实现：把一个在源任务（如 ImageNet 1000 类分类）上预训练好的模型，作为初始化权重，在目标任务（如猫狗二分类、医学影像分割）上继续训练。与&quot;从零开始训练（train from scratch）&quot;相比，微调能：</p><ul><li>显著减少所需训练数据量（从万级降到千级甚至百级）。</li><li>缩短训练时间（往往只需几个 epoch 即可收敛）。</li><li>提升最终精度，尤其在小数据集上效果显著。</li><li>降低过拟合风险（预训练特征本身有一定正则化作用）。</li></ul><p>本质上，微调把模型的优化起点从&quot;随机噪声&quot;挪到了&quot;已经学到良好低层特征&quot;的位置，使其更容易找到目标任务的好解。</p><h3><strong>7.5.2 微调有哪些不同方法？</strong></h3><table><tr><td><p><strong>方法</strong></p></td><td><p><strong>做法</strong></p></td><td><p><strong>适用场景</strong></p></td></tr><tr><td><p>特征提取（Linear Probing）</p></td><td><p>冻结全部预训练参数，只训练新加的分类头</p></td><td><p>数据量极少（&lt;1k）；任务与预训练高度相似；快速 baseline</p></td></tr><tr><td><p>浅层微调</p></td><td><p>冻结底层若干 block，只微调高层 + 分类头</p></td><td><p>中等数据量；任务与预训练较相似；显存有限</p></td></tr><tr><td><p>全网络微调</p></td><td><p>所有参数都参与训练，但用较小学习率</p></td><td><p>数据量充足（&gt;10k）；任务与预训练有一定差异</p></td></tr><tr><td><p>分层学习率（Discriminative LR）</p></td><td><p>不同层用不同 lr：底层 lr 小，顶层 lr 大</p></td><td><p>需要更精细控制（fastai、ULMFiT 风格）；NLP 任务常用</p></td></tr><tr><td><p>Adapter / LoRA</p></td><td><p>插入小的可训练模块，冻结主网络</p></td><td><p>大模型（LLM、ViT-L）；多任务部署；参数高效微调</p></td></tr><tr><td><p>Prompt Tuning / Prefix Tuning</p></td><td><p>只学习输入端的连续提示向量</p></td><td><p>超大规模 LLM；每任务参数 &lt; 1%</p></td></tr></table><p># PyTorch 典型微调：冻结底层 + 替换分类头<br />import torchvision.models as models<br />model = models.resnet50(weights=&quot;IMAGENET1K_V2&quot;)<br /><br /># 冻结所有层<br />for p in model.parameters():<br />    p.requires_grad = False<br /><br /># 替换最后的全连接（自动 requires_grad=True）<br />model.fc = nn.Linear(model.fc.in_features, num_classes)<br /><br /># 只优化未冻结的参数<br />optimizer = torch.optim.Adam(<br />    filter(lambda p: p.requires_grad, model.parameters()), lr=1e-3)</p><h3><strong>7.5.3 微调先冻结底层、训练顶层的原因？</strong></h3><p><strong>【特征层级理论】</strong></p><p>CNN 的特征学习呈现明显的层级性（Zeiler &amp; Fergus, 2014 的可视化工作）：</p><ul><li>浅层（第 1-2 层）：学到通用的低层特征（边缘、颜色块、纹理），这些特征在任何视觉任务中都通用。</li><li>中层（第 3-4 层）：学到部件级特征（眼睛、轮子、纹路图案），具有一定的领域通用性。</li><li>深层（最后几层）：学到任务特定的高层语义（&quot;猫脸&quot;、&quot;狗的姿态&quot;），与具体类别强相关。</li></ul><p><strong>【因此微调策略是：】</strong></p><ol><li>底层（通用特征层）：冻结或用极小学习率（如 1e-5）。它们已经&quot;学得很好&quot;，在小数据集上继续训练反而会破坏这些通用特征（catastrophic forgetting）。</li><li>中层：可适度微调，学习率介于底层和顶层之间。</li><li>顶层 + 分类头：完全替换（因为类别数变了）并用较大学习率（如 1e-3）训练。它们承担&quot;把通用特征映射到新任务标签空间&quot;的核心职责。</li></ol><p><strong>⚠️ 注意： </strong>错误做法警示：① 全网络用相同的大学习率（如 1e-2）微调——会&quot;震碎&quot;预训练权重，反而不如 train from scratch；② 完全不冻结也不调小学习率——容易过拟合小数据集。</p><h3><strong>7.5.4 不同数据集特性下如何微调？</strong></h3><p>这是一个经典的&quot;2×2 决策矩阵&quot;，按&quot;数据集规模&quot;和&quot;与预训练数据的相似度&quot;两个维度划分。</p><p><img src="/assets/img/dl-interview/img-21.png" /></p><p><em>图7-2 微调策略决策矩阵：根据数据规模和相似度选择策略</em></p><table><tr><td><p><strong>象限</strong></p></td><td><p><strong>场景</strong></p></td><td><p><strong>推荐策略</strong></p></td></tr><tr><td><p>Q1：数据多+相似</p></td><td><p>如：ImageNet 预训练 → 自建图像分类（数十类）</p></td><td><p>全网络微调；lr 中等（1e-4 ~ 1e-3）；这是最理想的情况</p></td></tr><tr><td><p>Q2：数据少+相似</p></td><td><p>如：ImageNet → 猫狗分类（1000 张）</p></td><td><p>冻结底层，只训练顶层；甚至直接 Linear Probing；强正则化（dropout、mixup、增强）防止过拟合</p></td></tr><tr><td><p>Q3：数据少+不同</p></td><td><p>如：ImageNet → 医学 X 光分类（500 张）</p></td><td><p>最棘手的情况；尝试微调早期/中期层（底层通用特征仍有用）；考虑半监督 / self-supervised 预训练替代 ImageNet 权重</p></td></tr><tr><td><p>Q4：数据多+不同</p></td><td><p>如：ImageNet → 卫星图像分割（10 万张）</p></td><td><p>可以全网络微调，但 lr 略大；也可以考虑 train from scratch，甚至先在领域数据上做 self-supervised 预训练</p></td></tr></table><h3><strong>7.5.5 目标检测中使用预训练模型的优劣？</strong></h3><p><strong>【优势】</strong></p><ul><li>收敛速度快：用 ImageNet 预训练的 backbone 启动 COCO 检测训练，收敛 epoch 数比 from scratch 少 4~6 倍。</li><li>小数据集精度更高：当检测数据集只有数千张图像时，预训练几乎是必须的。</li><li>降低硬件门槛：预训练初始化让模型对学习率、初始化等超参数更鲁棒。</li></ul><p><strong>【劣势】</strong></p><ul><li>架构受限：必须使用与预训练对齐的 backbone（ResNet、Swin 等），无法自由设计新结构。</li><li>域差距问题：ImageNet 是 RGB 自然图像，与 SAR、医学影像、深度图等差异大时，预训练特征可能反而是&quot;坏先验&quot;。</li><li>引入归纳偏置不可控：预训练模型对某些类别（如 ImageNet 中的犬科）有偏向，在长尾或类别极不平衡数据集上可能放大偏差。</li><li>训练范式被绑定：BN 的统计量来自 ImageNet 分布，迁移到新数据集时需要特别处理（如 frozen BN）。</li></ul><h3><strong>7.5.6 目标检测中如何从零开始训练（train from scratch）？</strong></h3><p><strong>【背景】</strong></p><p>何凯明等人在 &quot;Rethinking ImageNet Pre-training&quot;（2018）中证明：当训练数据充足、训练时间足够时，COCO 检测器从零开始训练能达到与预训练相当甚至略优的精度。这打破了&quot;必须 ImageNet 预训练&quot;的迷信。</p><p><strong>【关键技术】</strong></p><ol><li>Group Normalization 或 SyncBN：替代 frozen BN，因为 batch size 在检测任务中通常很小。GN 完全不依赖 batch 统计量，是 from scratch 训练的关键组件之一。</li><li>训练更久：from scratch 需要约 3~9 倍的 epoch 数（COCO 从 12 epoch 增至 72 epoch）。</li><li>正则化加强：mixup、cutmix、强数据增强、更大的 weight decay。</li><li>合理的初始化：He / Kaiming 初始化对深网络至关重要。</li></ol><p><strong>【典型代表】</strong></p><ul><li>DSOD（2017）：第一个论证小数据集也能 from scratch 训练检测器的工作，提出 deep supervision 等技巧。</li><li>Rethinking ImageNet Pre-training（He et al., 2018）：在 COCO 上系统对比，证明 from scratch 可行。</li><li>CenterNet / FCOS / DETR：现代 anchor-free 检测器，对预训练依赖度有所降低。</li></ul><p><strong>💡 提示： </strong>工程结论：① 数据量小 / 训练资源有限 → 用预训练（成本/收益最佳）；② 数据量大（&gt;10 万） / 域差异大 → 考虑 from scratch；③ 折中方案：在领域内数据上做 self-supervised 预训练（如 MAE、SimCLR），再微调，这是当前的主流范式。</p><h2><strong>7.6 自动化超参数搜索方法有哪些？</strong></h2><p>随着模型复杂度增加，手工调参的代价越来越大。自动化超参数优化（HPO/AutoML）工具能在给定预算下系统地搜索超参数空间。</p><p><img src="/assets/img/dl-interview/img-22.png" /></p><p><em>图7-3 四种主流超参数搜索方法的采样模式对比</em></p><h3><strong>7.6.1 网格搜索（Grid Search）</strong></h3><p>对每个超参数定义一个离散的候选值集合，遍历所有组合。例如 lr ∈ {1e-4, 1e-3, 1e-2}, bs ∈ {32, 64, 128} 共 9 个组合。</p><ul><li>优点：实现简单；可完整覆盖空间；易并行；结果可复现。</li><li>缺点：维度灾难——超参数个数增加，组合数指数爆炸；对每个维度的采样密度低。</li><li>适用：超参数 ≤ 3 个的小空间；快速验证 baseline。</li></ul><h3><strong>7.6.2 随机搜索（Random Search）</strong></h3><p>从每个超参数的分布中独立随机采样。Bergstra &amp; Bengio (2012) 证明：在相同试验次数下，随机搜索通常优于网格搜索，因为只有少数超参数真正重要，网格搜索把预算浪费在了不重要的维度上。</p><ul><li>优点：实现简单；对高维空间更友好；易并行。</li><li>缺点：仍然是无信息搜索，不利用已有评估结果。</li><li>实践：sklearn.model_selection.RandomizedSearchCV；注意学习率等用对数均匀采样：lr ~ LogUniform(1e-5, 1e-1)。</li></ul><h3><strong>7.6.3 贝叶斯优化（Bayesian Optimization）</strong></h3><p>基于高斯过程（GP）或 Tree Parzen Estimator（TPE）等代理模型，从已评估的样本中拟合&quot;超参数→指标&quot;的映射，通过采集函数（如 Expected Improvement, UCB）选择下一个最值得评估的超参数点，在&quot;探索（exploration）&quot;和&quot;利用（exploitation）&quot;间平衡。</p><ul><li>优点：采样效率高，少量试验即可逼近最优；适合训练昂贵的任务。</li><li>缺点：串行依赖（下一个点取决于前面所有点）；维度增加时高斯过程计算变慢；不利于并行化。</li><li>实现：Optuna（TPE 默认）、Ax/BoTorch（Meta）、scikit-optimize、Hyperopt。</li></ul><h3><strong>7.6.4 Hyperband / Successive Halving</strong></h3><p>核心思想：早期淘汰（early stopping）—— 先用很少的资源（如 1 epoch）评估很多配置，保留 top 1/η，再投入更多资源继续训练，逐轮淘汰，直到只剩最优配置。</p><ul><li>Successive Halving (SHA)：单轮逐步减半。</li><li>Hyperband：多次以不同 η 运行 SHA，自动权衡探索宽度和深度。</li><li>优点：充分利用部分训练曲线的信息，效率远高于随机搜索；天然支持并行。</li><li>缺点：要求&quot;中间指标能预测最终性能&quot;——对学习率冷启动差的任务（如 transformer warmup）可能误判。</li></ul><h3><strong>7.6.5 BOHB（Bayesian Optimization + Hyperband）</strong></h3><p>把贝叶斯优化和 Hyperband 结合：用 Hyperband 决定资源分配，用 BO（基于 TPE）决定采样下一个点。兼顾了&quot;高效利用早期评估&quot;和&quot;贝叶斯采样的信息引导&quot;，是 AutoML 的主流方法之一。</p><h3><strong>7.6.6 Population Based Training（PBT, DeepMind）</strong></h3><p>维护一个&quot;种群&quot;（如 16 个并行 worker），每个 worker 跑不同超参数。周期性地：① 表现差的 worker 复制表现好的 worker 的权重和超参数；② 对超参数做随机扰动（perturb）。相当于&quot;进化算法 + 边训练边调参&quot;。</p><ul><li>优点：超参数动态调整（如 lr 随训练阶段自动衰减）；适合 RL 等训练曲线波动大的任务。</li><li>缺点：需要充足的并行资源（一般 ≥ 16 个 worker）；实现复杂。</li><li>典型应用：AlphaStar 训练、Catch-Up Learning，Ray Tune 内置支持。</li></ul><h3><strong>7.6.7 总结：如何选择 HPO 方法？</strong></h3><table><tr><td><p><strong>场景</strong></p></td><td><p><strong>推荐方法</strong></p></td><td><p><strong>工具</strong></p></td></tr><tr><td><p>超参数 ≤ 3 个，每次训练快</p></td><td><p>网格搜索</p></td><td><p>sklearn</p></td></tr><tr><td><p>维度 4~10，每次训练几小时</p></td><td><p>随机搜索 / 贝叶斯优化</p></td><td><p>Optuna, Hyperopt</p></td></tr><tr><td><p>维度 4~10，每次训练 ≥ 1 天</p></td><td><p>Hyperband / BOHB</p></td><td><p>Ray Tune, HpBandSter</p></td></tr><tr><td><p>并行资源充足，且训练曲线波动大（如 RL）</p></td><td><p>PBT</p></td><td><p>Ray Tune</p></td></tr><tr><td><p>超大模型微调（如 LLM）</p></td><td><p>人工 + 少量贝叶斯优化</p></td><td><p>业内仍以经验为主</p></td></tr></table><p><strong>💡 提示： </strong>一个实用工作流：① 用 LR Range Test 定下学习率上限；② 用随机搜索粗扫 20 组配置，找到 top-3 配置的&quot;邻域&quot;；③ 用贝叶斯优化在邻域内精搜 30 组；④ 在最终模型上用更长 epoch 验证。</p><h2><strong>—— 全文完 ——</strong></h2><p>本文档系统梳理了深度学习面试常见的 7 大模块约 100 个问题，从神经网络基础到 GAN 和超参数调优，配合 20+ 张概念示意图。希望能成为读者面试准备和日常工作中的有用参考。技术内容会随研究进展更新，建议结合最新论文与开源代码持续学习。</p>
